# 知识库

## 基础项目推荐

搭建本地知识库风格的博客网站，核心需求通常包括：**Markdown 原生支持**、**本地部署便捷**、**内容管理高效**（文件/目录式）、**搜索/导航友好**，以及成熟的社区和生态。以下是 GitHub 上最成熟、适配性最强的项目推荐，按**使用场景和技术栈**分类，附核心特点和选型建议。

> ✅ **链接验证说明**：以下所有原项目的 GitHub 链接均经逐页验证，全部有效。其中 **Docute** 项目已停止积极维护（官方 README 标注），但仍可作为轻量备选参考。各分类同时新增了更多高质量选项，以满足不同技术栈和使用场景。

#### 一、静态站点生成器（博客+知识库首选，性能拉满）

将 Markdown 编译为静态 HTML，本地运行/部署极快，支持自定义主题和目录/导航结构，是最主流的选择。

##### 1. Hugo（⭐️ 77k+，Go）

[GitHub](https://github.com/gohugoio/hugo)

- **核心**：速度天花板（毫秒级构建千篇文档），完整 Markdown 支持，内置多语言、分类、标签、搜索（可集成 Algolia），主题生态丰富（[Hugo Themes](https://themes.gohugo.io/) 含大量知识库/文档主题）。
- **优势**：跨平台（Windows/macOS/Linux），单二进制文件无运行时依赖，本地 `hugo server` 启动，Markdown 实时热更新。
- **场景**：内容量大的本地知识库+博客（如上百篇技术笔记），追求极致性能和跨平台。

##### 2. VitePress（⭐️ 26k+，Vue+Vite）

[GitHub](https://github.com/vuejs/vitepress)

- **核心**：Vue 团队出品，基于 Vite，轻量、极速热更新，原生支持 Markdown（含 KaTeX 公式、代码高亮、自定义 Vue 组件），默认主题为极简知识库/文档风格（Vue3/Vite 官方文档使用）。
- **优势**：前端开发者友好，可在 Markdown 中嵌入 Vue 组件实现交互（如动态演示），配置极简，`vitepress dev` 秒开。
- **场景**：技术开发者的本地知识库（尤其前端），需嵌入交互组件，追求现代简洁界面。

##### 3. VuePress（⭐️ 23k+，Vue）

[GitHub](https://github.com/vuejs/vuepress)

- **核心**：VitePress 前身，生态更成熟，主题和插件更多（支持本地搜索、思维导图、评论等），支持 Markdown+Vue 组件，默认主题为文档风格。
- **优势**：稳定性高，适合需丰富插件的扩展场景（如图床、评论）。
- **场景**：同 VitePress，更适合需要大量插件的知识库/博客。

##### 4. MkDocs（⭐️ 16k+，Python）

[GitHub](https://github.com/mkdocs/mkdocs)

- **核心**：Python 生态静态文档生成器，配置极简（YAML），完整 Markdown 支持，**Material for MkDocs**（⭐️ 18k+）主题颜值极高、功能全（内置搜索、暗黑模式、目录导航、代码高亮、多语言）。
- **优势**：Python 开发者友好，`mkdocs serve` 启动，修改实时更新，Material 主题满足知识库多数需求。
- **场景**：非前端开发者（Python/数据）的本地知识库，追求开箱即用的高颜值文档风格。

##### 5. Docusaurus（⭐️ 42k+，React）

[GitHub](https://github.com/facebook/docusaurus)

- **核心**：Facebook 出品，基于 React，专为开源项目文档+博客设计，支持多语言、版本化文档、内置搜索（Algolia）、博客功能，主题简洁现代。
- **优势**：集成度高，一键生成文档+博客组合站点，适合版本管理的知识库（如技术文档不同版本）。
- **场景**：需版本化管理的技术知识库+博客，React 开发者友好。

##### 6. Quartz（⭐️ 11.7k+，TypeScript）

[GitHub](https://github.com/jackyzha0/quartz)

- **核心**：专为数字花园和双链笔记设计，支持与 Obsidian 深度集成，提供网络化思维、双向链接等知识管理功能。
- **优势**：原生支持本地 Markdown 文件，可直接使用 Obsidian 仓库作为内容源，生成知识网络站点。
- **场景**：笔记驱动的个人知识库，习惯 Obsidian 生态，希望将笔记发布为可交互的知识网络。

##### 7. Zola（⭐️ 16k+，Rust）

[GitHub](https://github.com/getzola/zola)

- **核心**：单二进制文件，内置 Sass 编译、图片处理、多语言支持，强调“开箱即用”，性能优异。
- **优势**：Rust 实现，构建速度接近 Hugo，配置比 Hugo 更简洁，主题生态逐步丰富。
- **场景**：追求高性能且偏好 Rust 生态的技术知识库，或对 Hugo 配置感到繁琐的用户。

##### 8. mdBook（⭐️ 21.5k+，Rust）

[GitHub](https://github.com/rust-lang/mdBook)

- **核心**：Rust 官方出品，类似 GitBook 但使用 Rust 实现，原生支持目录结构、搜索和代码高亮。
- **优势**：极简配置，输出专业的技术文档风格，支持将书籍输出为单个 HTML 文件或 PDF。
- **场景**：技术文档、电子书制作、需要严格章节结构的本地知识库。

##### 9. Eleventy（⭐️ 18k+，JavaScript）

[GitHub](https://github.com/11ty/eleventy)

- **核心**：轻量灵活，支持 Markdown、Liquid、Nunjucks 等多种模板引擎，零配置起步，插件生态丰富。
- **优势**：无强制前端框架，输出纯净 HTML，性能优秀，社区活跃。
- **场景**：希望自由选择模板引擎、不绑定特定框架的静态博客或知识库。

##### 10. Gatsby（⭐️ 55k+，React）

[GitHub](https://github.com/gatsbyjs/gatsby)

- **核心**：React 生态老牌 SSG，数据层（GraphQL）强大，支持从任意数据源拉取内容。
- **优势**：丰富的插件生态（图片优化、PWA、数据源等），适合内容驱动型站点。
- **场景**：需要从多个数据源（CMS、API、本地文件）聚合内容的大型知识库或博客。

##### 11. Next.js（⭐️ 139k+，React）

[GitHub](https://github.com/vercel/next.js)

- **核心**：混合渲染（SSG + SSR）的代表性框架，支持静态导出，同时可保留动态 API 路由。
- **优势**：React 生态最流行的框架之一，社区庞大，可轻松扩展为全栈应用。
- **场景**：需要部分动态交互（如评论、搜索后端）的大型知识库或混合站点。

##### 12. Astro（⭐️ 53k+，多框架）

[GitHub](https://github.com/withastro/astro)

- **核心**：创新的“岛屿架构”，按需加载客户端 JS，默认零 JS 输出，支持 React/Vue/Svelte 等多种组件。
- **优势**：性能极致，可在 Markdown 中嵌入任意 UI 框架组件，同时保持极低的运行时开销。
- **场景**：追求极致性能和现代化开发体验的知识库，需要灵活复用不同框架的组件。

#### 二、轻量无构建知识库（零配置，即时编辑）

无需编译构建，直接加载 Markdown 渲染为网页，修改文件刷新即生效，适合快速搭建本地知识库。

##### 1. Docsify（⭐️ 29k+，JavaScript）

[GitHub](https://github.com/docsifyjs/docsify)

- **核心**：零配置、无构建，一个 HTML 文件 + Markdown 目录即可生成知识库网站，支持侧边栏、搜索、目录导航、暗黑模式，体积极小。
- **优势**：本地运行只需打开 HTML 文件（或用 `docsify serve` 启动简易服务器），编辑 Markdown 即时生效。
- **场景**：快速搭建本地知识库，不想折腾配置，内容量适中。

##### 2. Docute（⭐️ 4.7k+，Vue）

[GitHub](https://github.com/egoist/docute)

- **核心**：类似 Docsify，零构建、轻量，基于 Vue，支持 Markdown 即时渲染，界面更现代，支持自定义组件。
- **优势**：Vue 驱动，交互更流畅，配置稍灵活。
- **注意**：该项目已停止维护，但仍可正常使用，适合喜欢 Vue 生态的轻量本地知识库。

##### 3. SilverBullet（⭐️ 4k+，Go + TypeScript）

[GitHub](https://github.com/silverbulletmd/silverbullet)

- **核心**：可编程的个人知识管理平台，基于 Markdown 文件，支持双向链接、任务管理、Lua 脚本扩展。
- **优势**：兼具笔记软件与文档站点能力，提供在线编辑器，修改保存即刷新站点。
- **场景**：需要本地编辑+即时预览的知识库，喜欢 Notion 风格但希望基于纯文本文件。

##### 4. Raneto（⭐️ 2.7k+，Node.js）

[GitHub](https://github.com/gilbitron/Raneto)

- **核心**：纯文件驱动的知识库平台，提供内置浏览器编辑器、登录保护、全文搜索，无需数据库。
- **优势**：轻量级 Node.js 应用，部署简单，适合团队内部快速搭建文档中心。
- **场景**：需要简单后台编辑器（非技术人员也可编辑）的本地/内网知识库。

#### 三、动态知识库/博客系统（带后台管理，支持数据库）

需要可视化后台管理、用户评论、动态内容发布（非纯文件管理），本地部署需搭配数据库，功能更丰富。

##### 1. Wiki.js（⭐️ 31k+，Node.js）

[GitHub](https://github.com/Requarks/wiki)

- **核心**：开源 Wiki，支持 Markdown 编辑、可视化后台、多存储方式（本地文件、Git、MySQL/PostgreSQL）、内置搜索、多语言、权限管理，可作知识库+博客。
- **优势**：兼具文件与数据库优势，支持内容同步到 Git（如 GitHub/Gitee），本地部署简单（Docker 一键启动）。
- **场景**：需后台管理和权限控制的团队/个人本地知识库，支持多人协作编辑。

##### 2. Ghost（⭐️ 46k+，Node.js）

[GitHub](https://github.com/TryGhost/Ghost)

- **核心**：开源博客平台，专注内容创作，美观的后台管理界面，支持 Markdown 编辑、自定义主题、标签/分类，性能优秀，本地部署需 MySQL。
- **优势**：博客体验极致，主题生态丰富。
- **场景**：想要高颜值动态博客，附带简单知识库功能。

##### 3. Typecho（⭐️ 6.8k+，PHP）

[GitHub](https://github.com/typecho/typecho)

- **核心**：轻量 PHP 博客系统，体积仅几 MB，本地部署需 PHP+MySQL，支持 Markdown 插件，插件生态丰富（搜索、评论、知识库导航等）。
- **优势**：极度轻量，适合低配服务器/本地环境，上手简单。
- **场景**：轻量动态博客+小型知识库，PHP 环境用户。

##### 4. Gollum（⭐️ 13.7k+，Ruby）

[GitHub](https://github.com/gollum/gollum)

- **核心**：Git 驱动的 Wiki 系统，与 GitHub/GitLab Wiki 兼容，支持本地编辑与 Web 界面双模式。
- **优势**：无缝集成 Git 版本控制，适合技术团队文档协作，支持多种标记语言。
- **场景**：开发团队内部知识库，希望像管理代码一样管理文档。

##### 5. HedgeDoc（⭐️ 5.4k+，TypeScript）

[GitHub](https://github.com/hedgedoc/hedgedoc)

- **核心**：实时协作 Markdown 编辑器（原 CodiMD / HackMD 社区版），支持多人同时编辑、版本历史、幻灯片模式。
- **优势**：类似 Google Docs 的协作体验，支持匿名编辑和权限控制，适合团队实时会议记录或文档共创。
- **场景**：团队需要实时协作编写知识文档，且希望保留 Markdown 纯文本格式。

##### 6. Docmost（⭐️ 19.7k+，TypeScript）

[GitHub](https://github.com/docmost/docmost)

- **核心**：开源的 Confluence/Notion 替代品，支持实时协作、Draw.io/Mermaid 图表、权限管理、页面历史。
- **优势**：功能丰富，界面现代，支持自托管，团队知识管理利器。
- **场景**：需要类似 Notion 体验但希望完全自托管的团队知识库。

##### 7. MM-Wiki（轻量级）

[GitHub](https://github.com/phachon/mm-wiki)

- **核心**：专为团队设计的轻量级企业知识库，基于 Go 开发，单二进制文件部署，支持 LDAP 和细粒度权限控制。
- **优势**：部署极简，资源占用低，特别适合内网环境，支持 Markdown 编辑和全文搜索。
- **场景**：中小团队在内网搭建快速、轻量的知识库系统。

#### 四、选型建议（已整合新增项目）

1. **纯本地知识库，追求简单快捷**：选 **Docsify**（零配置）、**MkDocs+Material**（高颜值、开箱即用）或 **Raneto**（内置编辑器）。
2. **知识管理/笔记驱动的知识库**：选 **Quartz**（Obsidian 深度集成）、**SilverBullet**（可编程 PKM）。
3. **技术开发者，需嵌入交互组件**：选 **VitePress/VuePress**（Vue）、**Docusaurus**（React）或 **Astro**（多框架，性能极致）。
4. **内容量大，追求极致性能**：选 **Hugo**（Go）、**Zola**（Rust）或 **mdBook**（Rust，技术文档首选）。
5. **需要后台管理 / 多人协作**：选 **Wiki.js**（知识库）、**Docmost**（团队文档）、**Ghost**（博客）或 **HedgeDoc**（实时协作）。
6. **非前端开发者（Python/PHP/Ruby）**：选 **MkDocs**（Python）、**Typecho**（PHP）、**Gollum**（Ruby）或 **Zola**（Rust 单文件）。
7. **大型动态混合站点**：选 **Next.js** 或 **Gatsby**（SSG + SSR 混合渲染，数据源丰富）。
8. **团队内网轻量部署**：选 **MM-Wiki**（Go 单文件，LDAP 支持）。

## VitePress

Vue 官方出品的静态文档站点生成器。

### 核心优势

1. **1:1 对齐示例效果**：自带侧边栏导航、右侧文章目录、深色/浅色模式切换、全局快捷键搜索、版本切换、国际化等功能，与你截图中的能力完全一致。
2. **极致轻量，零门槛上手**：基于 Markdown 编写内容，无需手写前端页面；基于 Vite，启动和打包极快，开发体验极佳。
3. **灵活可扩展**：支持嵌入 Vue 组件自定义页面，也可通过插件扩展功能。
4. **部署零成本**：生成纯静态 HTML，可托管至任何静态服务。

### 执行初始化命令

bash

```
mkdir my-knowledge-base && cd my-knowledge-base
npm init -y
pnpm add -D vitepress@next
pnpm vitepress init
```

> VitePress 是仅 ESM 的软件包。不要使用 `require()` 导入，确保 `package.json` 包含 `"type": "module"`，或将相关文件扩展名改为 `.mjs`/`.mts`。详情参考 [Vite 故障排除指南](http://vitejs.dev/guide/troubleshooting.html#this-package-is-esm-only)。异步 CJS 上下文中，可用 `await import('vitepress')` 代替。

### 官方向导选项解析

```
┌  Welcome to VitePress!
│
◇  Where should VitePress initialize the config?
│  ./docs
│	指定核心配置文件夹（.vitepress）的存放路径，默认为 ./docs（最终路径：docs/.vitepress/config.ts），存放全局配置（导航栏、侧边栏、主题等）。
◇  Where should VitePress look for your markdown files?
│  ./docs
│	指定知识库 Markdown 文档的存放根目录。所有 .md 文件放于 ./docs 下，VitePress 自动扫描生成网页。
◇  Site title:
│  My Awesome Project
│	网站全局标题（显示在浏览器标签、首页头部、SEO 标题等），后续可在 config.ts 中修改（title 字段）。
◇  Site description:
│  A VitePress Site
│	网站描述（用于 SEO 和搜索结果展示）。
◇  Theme:
│  Default Theme
│	选择基础主题。
◇  Use TypeScript for config and theme files?
│  Yes
│	配置文件使用 TypeScript（.ts）还是 JavaScript（.js）。
◇  Add VitePress npm scripts to package.json?
│  Yes
│	自动添加 VitePress 快捷命令（如 docs:dev、docs:build），无需手动记复杂命令。新手建议选 Yes。
◇  Add a prefix for VitePress npm scripts?
│  Yes
│	是否给 npm 命令加前缀（避免与项目中其他脚本冲突）。纯知识库可不加，加前缀更规范。
◇  Prefix for VitePress npm scripts:
│  docs
│	指定命令前缀，最终命令为 docs:dev、docs:build。
└  Done! Now run pnpm run docs:dev and start writing.
```
