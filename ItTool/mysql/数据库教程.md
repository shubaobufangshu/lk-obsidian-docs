### 零初始化
#### 初始化 MySQL 数据目录
这是一个用于 **初始化 MySQL 数据目录** 的命令，通常在首次安装 MySQL 后使用。
```bash
mysqld --defaults-file="url\my.ini" --initialize --console
```
##### 参数解释
+ `mysqld`
  + **作用**：MySQL 服务器的可执行程序
  + **说明**：用于启动 MySQL 数据库服务的主程序
+ `--defaults-file="url\my.ini"`
  
  + **作用**：指定配置文件路径
  + **详细说明**：
    - 告诉 MySQL 使用指定位置的配置文件，而不是默认位置的配置文件
    - 使用此参数可以精确指定配置文件路径
    - **文件内容示例**：
  
    ```bash
    [mysqld]
    basedir=C:/Users/lk/Desktop/MySQLServer5.7
    datadir=C:/Users/lk/Desktop/MySQLServer5.7/data
    port=3306
    ```
+ `--initialize`
  + **作用**：初始化 MySQL 数据目录
  + **详细说明**：
    - **安全模式初始化**（MySQL 5.7.6+）：生成一个随机的 root 初始密码
    - 创建系统数据库（mysql, sys, information_schema, performance_schema）
    - 创建数据文件结构和表
    - **重要**：只能在新安装时执行一次！如果数据目录已存在会报错
+ `--console`
  + **作用**：将日志输出到控制台
  + **详细说明**：
    - 让 MySQL 将日志信息直接显示在命令行窗口中
    - 方便查看初始化过程的状态和结果
    - 特别是查看 **生成的 root 初始密码**

### 管理员配置

#### MySQL 权限体系基础
MySQL 的权限是**层级化**的，不同粒度的权限适配不同场景的管理员，先理清权限层级能避免权限分配过宽 / 过窄：
| 权限层级   | 作用范围                 | 适用场景                 |
| ---------- | ------------------------ | ------------------------ |
| 全局级别   | 整个 MySQL 实例（`*.*`） | 超级管理员（替代 root）  |
| 数据库级别 | 指定数据库（`mydb.*`）   | 业务数据库专属管理员     |
| 表级别     | 指定数据库的指定表       | 单表精细化管理（极少用） |
| 列级别     | 指定表的指定列           | 敏感字段管控（极少用）   |
常用管理员核心权限关键词：
- `ALL PRIVILEGES`：所有操作权限（不含授权权限）
- `GRANT OPTION`：授权权限（允许该用户给其他用户分配权限）
- `SELECT/INSERT/UPDATE/DELETE`：基础数据操作权限
- `CREATE/DROP/ALTER`：库 / 表结构修改权限
- `PROCESS/SUPER`：系统管理权限（查看进程、修改系统变量）
#### 核心操作：创建管理员用户
生产环境**严禁直接使用 root 用户做日常管理**，需根据场景创建专用管理员，以下是 3 种典型场景：
##### 场景 1：创建超级管理员（替代 root，全局权限）
适合需要管理整个 MySQL 实例的管理员，建议限制访问 IP（而非任意主机）：
```sql
-- 1. 创建用户：格式 CREATE USER '用户名'@'访问来源' IDENTIFIED BY '密码';
-- 访问来源：%=任意主机，192.168.1.%=指定网段，localhost=仅本地
CREATE USER 'db_super_admin'@'192.168.1.%' IDENTIFIED BY 'Super@123456';
-- 2. 赋予全局所有权限，并允许该用户给其他用户授权（WITH GRANT OPTION）
GRANT ALL PRIVILEGES ON *.* TO 'db_super_admin'@'192.168.1.%' WITH GRANT OPTION;
-- 3. 刷新权限（修改权限后必须执行，使配置立即生效）
FLUSH PRIVILEGES;
```
##### 场景 2：创建数据库级管理员（仅管理指定库）
适合业务管理员，仅能操作指定数据库（如电商库`ecommerce_db`），遵循 “最小权限原则”：
```sql
-- 创建用户（仅允许从192.168.1.100这台机器访问，更安全）
CREATE USER 'ecom_admin'@'192.168.1.100' IDENTIFIED BY 'Ecom@654321';
-- 赋予ecommerce_db数据库的所有权限（不含全局权限）
GRANT ALL PRIVILEGES ON ecommerce_db.* TO 'ecom_admin'@'192.168.1.100';
-- 如需允许该用户给其他用户分配ecommerce_db的权限，添加授权权限
-- GRANT ALL PRIVILEGES ON ecommerce_db.* TO 'ecom_admin'@'192.168.1.100' WITH GRANT OPTION;
FLUSH PRIVILEGES;
```
##### 场景 3：创建只读管理员（仅查询权限）
适合数据报表、审计类管理员，无修改 / 删除权限，避免误操作：
```sql
CREATE USER 'read_only_admin'@'%' IDENTIFIED BY 'Read@123456';
-- 仅赋予全局只读权限（SELECT）
GRANT SELECT ON *.* TO 'read_only_admin'@'%';
FLUSH PRIVILEGES;
```
#### 管理员密码管理
##### 修改管理员密码
```sql
-- 方式1：ALTER USER（MySQL 8.0+推荐）
ALTER USER 'db_super_admin'@'192.168.1.%' IDENTIFIED BY 'NewSuper@654321';
-- 方式2：SET PASSWORD（兼容MySQL 5.7及以下）
SET PASSWORD FOR 'ecom_admin'@'192.168.1.100' = 'NewEcom@123456';  -- MySQL 8.0
-- MySQL 5.7需加PASSWORD()函数：SET PASSWORD FOR 'ecom_admin'@'192.168.1.100' = PASSWORD('NewEcom@123456');

FLUSH PRIVILEGES;
```
#####  重置忘记的管理员密码
若管理员密码丢失，通过`root`用户重置：
```sql
-- 重置read_only_admin的密码
ALTER USER 'read_only_admin'@'%' IDENTIFIED BY 'ResetRead@654321';
FLUSH PRIVILEGES;
```
##### 强制密码定期过期（MySQL 8.0+）
提升安全性，要求管理员定期改密码：
```sql
-- 单个用户：设置db_super_admin密码90天后过期
ALTER USER 'db_super_admin'@'192.168.1.%' PASSWORD EXPIRE INTERVAL 90 DAY;
-- 全局策略：所有用户密码默认90天过期
SET GLOBAL default_password_lifetime = 90;
FLUSH PRIVILEGES;
```

```sql
-- 输出示例：GRANT ALL PRIVILEGES ON *.* TO `db_super_admin`@`192.168.1.%` WITH GRANT OPTION
```

##### 回收管理员权限

若权限分配错误或需降级权限，可精准回收：

```sql
-- 回收db_super_admin的删除权限（保留其他权限）
REVOKE DELETE ON *.* FROM 'db_super_admin'@'192.168.1.%';

-- 回收ecom_admin的授权权限（禁止其给他人分配权限）
REVOKE GRANT OPTION ON ecommerce_db.* FROM 'ecom_admin'@'192.168.1.100';

-- 回收read_only_admin的所有权限（慎用）
REVOKE ALL PRIVILEGES ON *.* FROM 'read_only_admin'@'%';

FLUSH PRIVILEGES;
```

#### 删除管理员用户

当管理员离职或权限不再需要时，彻底删除用户：

```sql
-- 删除指定管理员用户
DROP USER 'read_only_admin'@'%';
DROP USER 'ecom_admin'@'192.168.1.100';

FLUSH PRIVILEGES;
```

#### 安全配置教程

1. **严格限制访问来源**：避免使用`%`（任意主机），优先指定具体 IP / 网段（如`192.168.1.%`）。

2. 强化密码策略

   ```sql
   -- 设置密码强度为最高（STRONG），要求包含大小写、数字、特殊字符
   SET GLOBAL validate_password_policy = STRONG;
   -- 密码最小长度设为10位
   SET GLOBAL validate_password_length = 10;
   FLUSH PRIVILEGES;
   ```

3. **定期审计权限**：每月执行`SHOW GRANTS FOR '用户名'@'来源';`检查权限是否合规。

4. 限制并发连接数：防止单管理员占用过多连接：

   ```sql
   ALTER USER 'db_super_admin'@'192.168.1.%' WITH MAX_USER_CONNECTIONS 10;
   FLUSH PRIVILEGES;
   ```

5. 禁用空密码 / 弱密码用户

   ```sql
   -- 删除所有空密码用户
   DELETE FROM mysql.user WHERE authentication_string = '';
   FLUSH PRIVILEGES;
   ```

## sql教程

### SQL 分类

| 分类         | 英文缩写 | 核心操作                        | 用途                                    |
| ------------ | -------- | ------------------------------- | --------------------------------------- |
| 数据查询语言 | DQL      | SELECT                          | 查询数据库数据                          |
| 数据操作语言 | DML      | INSERT、UPDATE、DELETE          | 操作数据库中的数据                      |
| 数据定义语言 | DDL      | CREATE、ALTER、DROP、TRUNCATE   | 定义 / 修改数据库对象（库、表、索引等） |
| 数据控制语言 | DCL      | GRANT、REVOKE、COMMIT、ROLLBACK | 权限管理、事务控制                      |
| 事务控制语言 | TCL      | COMMIT、ROLLBACK、SAVEPOINT     | 事务的提交、回滚                        |

### 一、基础语法（核心）

#### 1. 数据查询语言（DQL）—— SELECT

##### 1.1 基础查询

```sql
-- 1. 查询所有列
SELECT * FROM 表名;

-- 2. 查询指定列（多个列用逗号分隔）
SELECT 列名1, 列名2 FROM 表名;

-- 3. 列别名（AS 可省略）
SELECT 列名1 AS 别名1, 列名2 别名2 FROM 表名;

-- 4. 去重查询（DISTINCT）
SELECT DISTINCT 列名 FROM 表名;

-- 5. 计算列（支持 +-*/ 运算，NULL 参与运算结果为 NULL）
SELECT 列名1 + 列名2 AS 计算结果 FROM 表名;
-- 示例：查询学生总分（语文+数学+英语）
SELECT id, name, chinese + math + english AS total_score FROM student;
```

##### 1.2 条件筛选（WHERE）

支持的运算符：

- 比较运算符：`=`、`<>`/`!=`（不等于）、`>`、`<`、`>=`、`<=`
- 逻辑运算符：`AND`（且）、`OR`（或）、`NOT`（非）
- 范围 / 集合：`BETWEEN...AND...`（区间）、`IN(值1,值2...)`（集合匹配）
- 模糊查询：`LIKE`（`%` 匹配任意字符，`_` 匹配单个字符）
- 空值判断：`IS NULL`/`IS NOT NULL`（**不能用 = NULL**）

```sql
-- 示例：查询语文分数大于80且数学分数大于90的学生
SELECT name, chinese, math FROM student 
WHERE chinese > 80 AND math > 90;

-- 示例：查询分数在80-90之间的学生（包含边界）
SELECT name, english FROM student 
WHERE english BETWEEN 80 AND 90;

-- 示例：查询班级为1班或3班的学生
SELECT name, class FROM student 
WHERE class IN (1, 3);

-- 示例：查询姓名以“张”开头的学生（模糊查询）
SELECT name FROM student 
WHERE name LIKE '张%';

-- 示例：查询语文分数为空的学生
SELECT name FROM student 
WHERE chinese IS NULL;
```

##### 1.3 排序（ORDER BY）

```sql
-- 语法：ORDER BY 列名 [ASC（升序，默认）| DESC（降序）]
SELECT name, math FROM student 
ORDER BY math DESC; -- 按数学分数降序

-- 多列排序：先按班级升序，再按总分降序
SELECT name, class, total_score FROM student 
ORDER BY class ASC, total_score DESC;
```

##### 1.4 分页查询（LIMIT）

不同数据库分页语法略有差异，MySQL 用 `LIMIT`，Oracle 用 `ROWNUM`，SQL Server 用 `TOP`/`OFFSET`。

```sql
-- MySQL 分页：LIMIT 起始索引, 每页条数（起始索引从0开始）
-- 示例：查询第2页数据，每页10条（索引从10开始，取10条）
SELECT * FROM student LIMIT 10, 10;

-- 简化写法：取前N条（起始索引省略，默认0）
SELECT * FROM student LIMIT 5; -- 取前5条
```

##### 1.5 分组聚合（GROUP BY + HAVING）

**聚合函数**：`COUNT()`（计数）、`SUM()`（求和）、`AVG()`（平均值）、`MAX()`（最大值）、`MIN()`（最小值）。

- `WHERE` 筛选**原始数据**，`HAVING` 筛选**分组后的数据**。

```sql
-- 示例1：按班级分组，统计每个班级的学生人数
SELECT class, COUNT(*) AS student_count FROM student 
GROUP BY class;

-- 示例2：按班级分组，筛选出平均分数大于85的班级
SELECT class, AVG(total_score) AS avg_score FROM student 
GROUP BY class 
HAVING avg_score > 85;

-- 示例3：COUNT 细节（COUNT(*) 统计所有行，COUNT(列名) 排除NULL值）
SELECT COUNT(*) AS 总人数, COUNT(chinese) AS 语文有分数人数 FROM student;
```

#### 2. 数据操作语言（DML）

##### 2.1 插入数据（INSERT）

```sql
-- 方式1：指定列插入（推荐，列顺序可与表结构不一致）
INSERT INTO 表名(列名1, 列名2, 列名3) 
VALUES (值1, 值2, 值3);

-- 方式2：插入所有列（列顺序必须与表结构一致）
INSERT INTO 表名 
VALUES (值1, 值2, 值3, ...);

-- 方式3：批量插入（高效，减少IO）
INSERT INTO 表名(列名1, 列名2) 
VALUES (值1, 值2), (值3, 值4), (值5, 值6);

-- 示例：插入一条学生数据
INSERT INTO student(name, class, chinese, math) 
VALUES ('李四', 2, 88, 92);
```

##### 2.2 更新数据（UPDATE）

```sql
-- 语法：UPDATE 表名 SET 列名1=值1, 列名2=值2 [WHERE 条件]
-- 注意：无WHERE条件会更新表中所有行！
UPDATE student 
SET math = 95, chinese = 90 
WHERE id = 1; -- 更新id=1的学生的数学和语文分数
```

##### 2.3 删除数据（DELETE）

```sql
-- 语法：DELETE FROM 表名 [WHERE 条件]
-- 注意：无WHERE条件会删除表中所有行（可回滚）
DELETE FROM student 
WHERE id = 2; -- 删除id=2的学生数据

-- 清空表（两种方式）
DELETE FROM student; -- 逐行删除，可回滚，保留自增序列
TRUNCATE TABLE student; -- 清空数据并重置自增序列，不可回滚（效率更高）
```

#### 3. 数据定义语言（DDL）

##### 3.1 数据库操作

```sql
-- 1. 创建数据库（指定字符集和排序规则，推荐utf8mb4）
CREATE DATABASE IF NOT EXISTS 数据库名 
DEFAULT CHARACTER SET utf8mb4 
DEFAULT COLLATE utf8mb4_general_ci;

-- 2. 查看数据库
SHOW DATABASES; -- 查看所有数据库
SHOW CREATE DATABASE 数据库名; -- 查看数据库创建详情

-- 3. 使用数据库
USE 数据库名;

-- 4. 修改数据库（修改字符集）
ALTER DATABASE 数据库名 
DEFAULT CHARACTER SET utf8mb4;

-- 5. 删除数据库（谨慎！）
DROP DATABASE IF EXISTS 数据库名;
```

##### 3.2 表操作

###### 3.2.1 创建表（CREATE TABLE）

```sql
-- 语法
CREATE TABLE IF NOT EXISTS 表名 (
    列名1 数据类型 [约束] [COMMENT '注释'],
    列名2 数据类型 [约束] [COMMENT '注释'],
    ...
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT '表注释';

-- 示例：创建学生表
CREATE TABLE IF NOT EXISTS student (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '学生ID（主键，自增）',
    name VARCHAR(50) NOT NULL COMMENT '姓名',
    class INT COMMENT '班级',
    gender CHAR(1) DEFAULT '男' COMMENT '性别',
    chinese INT DEFAULT 0 COMMENT '语文分数',
    math INT DEFAULT 0 COMMENT '数学分数',
    create_time DATETIME COMMENT '创建时间'

```

MySQL 常用数据类型：

| 类型分类 | 常用类型                            | 用途                           |
| -------- | ----------------------------------- | ------------------------------ |
| 数值型   | INT、BIGINT、FLOAT、DOUBLE、DECIMAL | 整数、小数（DECIMAL 用于金额） |
| 字符串型 | VARCHAR、CHAR、TEXT                 | 变长字符串、定长字符串、大文本 |
| 日期型   | DATE、TIME、DATETIME、TIMESTAMP     | 日期、时间、日期时间、时间戳   |

###### 3.2.2 查看表

```sql
SHOW TABLES; -- 查看当前数据库所有表
DESC 表名; -- 查看表结构（列名、类型、约束等）
SHOW CREATE TABLE 表名; -- 查看表创建详情
```

###### 3.2.3 修改表（ALTER TABLE）

```sql
-- 1. 添加列
ALTER TABLE 表名 ADD 列名 数据类型 [约束] [COMMENT '注释'];
-- 示例：添加英语分数列
ALTER TABLE student ADD english INT DEFAULT 0 COMMENT '英语分数';

-- 2. 修改列（类型/约束/注释）
ALTER TABLE 表名 MODIFY 列名 新数据类型 [约束];
-- 示例：修改姓名列长度为60
ALTER TABLE student MODIFY name VARCHAR(60) NOT NULL;

-- 3. 修改列名
ALTER TABLE 表名 CHANGE 旧列名 新列名 数据类型 [约束];
-- 示例：将gender列改为sex
ALTER TABLE student CHANGE gender sex CHAR(1) DEFAULT '男';

-- 4. 删除列
ALTER TABLE 表名 DROP 列名;
-- 示例：删除create_time列

-- 5. 修改表名
ALTER TABLE 表名 RENAME TO 新表名;
```

###### 3.2.4 删除表

```sql
-- 谨慎！删除表结构和所有数据
DROP TABLE IF EXISTS 表名;
```

#### 4. 数据控制语言（DCL）

##### 4.1 权限管理

```sql
-- 1. 授予权限（给用户授予数据库/表的操作权限）
-- 语法：GRANT 权限列表 ON 数据库名.表名 TO '用户名'@'主机地址' IDENTIFIED BY '密码';
-- 示例：给test用户授予studentdb数据库所有表的查询/插入权限
GRANT SELECT, INSERT ON studentdb.* TO 'test'@'localhost' IDENTIFIED BY '123456';

-- 2. 撤销权限
REVOKE 权限列表 ON 数据库名.表名 FROM '用户名'@'主机地址';
-- 示例：撤销test用户的插入权限
REVOKE INSERT ON studentdb.* FROM 'test'@'localhost';

-- 3. 刷新权限
FLUSH PRIVILEGES;
```

##### 4.2 事务控制

```sql
-- 开启事务（关闭自动提交）
START TRANSACTION;

-- 执行DML操作（如INSERT/UPDATE/DELETE）
UPDATE student SET math = 100 WHERE id = 1;
DELETE FROM student WHERE id = 3;

-- 提交事务（所有操作生效）
COMMIT;

-- 回滚事务（撤销所有未提交的操作）
ROLLBACK;

-- 保存点（回滚到指定位置）
SAVEPOINT sp1; -- 创建保存点
ROLLBACK TO sp1; -- 回滚到保存点
```

### 二、高级查询

#### 1. 联表查询（JOIN）

用于多表关联查询，核心是通过**关联字段**（通常是主键 / 外键）连接多个表。

| 连接类型   | 说明                                                   |
| ---------- | ------------------------------------------------------ |
| INNER JOIN | 内连接：只返回两表匹配的行（默认 JOIN）                |
| LEFT JOIN  | 左连接：返回左表所有行，右表匹配不到则为 NULL          |
| RIGHT JOIN | 右连接：返回右表所有行，左表匹配不到则为 NULL          |
| FULL JOIN  | 全连接：返回两表所有行（MySQL 不直接支持，需用 UNION） |

```sql
-- 先创建两张关联表：学生表（student）、成绩表（score）
-- student：id(主键)、name、class
-- score：id、student_id(外键关联student.id)、subject、score

-- 1. 内连接：查询有成绩的学生及其分数
SELECT s.name, sc.subject, sc.score 
FROM student s
INNER JOIN score sc 
ON s.id = sc.student_id;

-- 2. 左连接：查询所有学生（包括无成绩的）及其分数
SELECT s.name, sc.subject, sc.score 
FROM student s
LEFT JOIN score sc 
ON s.id = sc.student_id;

-- 3. 右连接：查询所有成绩（包括无对应学生的）
SELECT s.name, sc.subject, sc.score 
FROM student s
RIGHT JOIN score sc 
ON s.id = sc.student_id;

-- 4. 多表连接：新增课程表（course），查询学生-成绩-课程信息
SELECT s.name, sc.score, c.course_name 
FROM student s
JOIN score sc ON s.id = sc.student_id
JOIN course c ON sc.course_id = c.id;
```

#### 2. 子查询

嵌套在其他 SQL 中的查询语句，分为**标量子查询**（返回单个值）、**列子查询**（返回一列）、**行子查询**（返回一行）、**表子查询**（返回多行多列）。

```sql
-- 示例1：标量子查询（查询数学分数最高的学生姓名）
SELECT name FROM student 
WHERE math = (SELECT MAX(math) FROM student);

-- 示例2：列子查询（查询1班或2班的学生成绩）
SELECT * FROM score 
WHERE student_id IN (SELECT id FROM student WHERE class IN (1,2));

-- 示例3：表子查询（查询总分大于200的学生，子查询作为临时表）
SELECT t.name, t.total_score 
FROM (SELECT name, chinese+math+english AS total_score FROM student) t
WHERE t.total_score > 200;
```

#### 3. 合并查询（UNION / UNION ALL）

用于合并多个 SELECT 的结果集，要求列数、列类型一致。

- `UNION`：去重（自动删除重复行）
- `UNION ALL`：不去重（效率更高）

```sql
-- 示例：合并1班和2班的学生名单
SELECT name, class FROM student WHERE class = 1
UNION ALL
SELECT name, class FROM student WHERE class = 2;
```

### 四、约束（Constraint）

约束用于保证数据的完整性和一致性，常用约束如下：

| 约束类型 | 关键字         | 说明                                           |
| -------- | -------------- | ---------------------------------------------- |
| 主键约束 | PRIMARY KEY    | 唯一标识行，非空且唯一（一张表只能有一个主键） |
| 自增约束 | AUTO_INCREMENT | 主键自动递增（MySQL 专属）                     |
| 非空约束 | NOT NULL       | 列值不能为 NULL                                |
| 唯一约束 | UNIQUE         | 列值唯一（可多个 NULL）                        |
| 默认约束 | DEFAULT        | 列值的默认值                                   |
| 外键约束 | FOREIGN KEY    | 关联另一张表的主键，保证数据引用完整性         |
| 检查约束 | CHECK          | 限制列值的范围（MySQL 8.0 + 支持）             |

```sql
-- 示例：创建带外键的成绩表
CREATE TABLE score (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '成绩ID',
    student_id INT NOT NULL COMMENT '学生ID',
    course_id INT NOT NULL COMMENT '课程ID',
    score INT CHECK (score >= 0 AND score <= 100) COMMENT '分数（0-100）',
    -- 外键约束：关联学生表
    FOREIGN KEY (student_id) REFERENCES student(id) ON DELETE CASCADE,
    -- 外键约束：关联课程表
    FOREIGN KEY (course_id) REFERENCES course(id) ON DELETE RESTRICT
) COMMENT '成绩表';

-- 外键行为说明：
-- ON DELETE CASCADE：主表行删除时，从表关联行也删除
-- ON DELETE RESTRICT：主表行有从表关联时，禁止删除
-- ON DELETE SET NULL：主表行删除时，从表关联列设为NULL（需从表列允许NULL）
```

### 五、索引（Index）

索引是提升查询效率的核心机制，相当于数据库的 “目录”，避免全表扫描。

#### 1. 索引分类

| 索引类型 | 关键字             | 说明                                 |
| -------- | ------------------ | ------------------------------------ |
| 主键索引 | PRIMARY KEY        | 主键自动创建，唯一且非空             |
| 唯一索引 | UNIQUE             | 列值唯一（可 NULL）                  |
| 普通索引 | INDEX              | 无唯一性限制，提升查询效率           |
| 复合索引 | INDEX (列 1, 列 2) | 多个列组合的索引（遵循最左匹配原则） |
| 全文索引 | FULLTEXT           | 用于大文本字段的模糊查询（如 TEXT）  |

#### 2. 索引操作

```sql
-- 1. 创建索引
-- 普通索引
CREATE INDEX idx_student_class ON student(class);
-- 唯一索引
CREATE UNIQUE INDEX idx_student_name ON student(name);
-- 复合索引
CREATE INDEX idx_score_stu_course ON score(student_id, course_id);

-- 2. 查看索引
SHOW INDEX FROM 表名;

-- 3. 删除索引
DROP INDEX 索引名 ON 表名;
-- 示例：删除学生表的班级索引
DROP INDEX idx_student_class ON student;
```

#### 3. 索引使用原则

- 适合加索引的场景：查询频繁的列、WHERE 条件列、JOIN 关联列。
- 不适合加索引的场景：更新频繁的列、数据量小的表、重复值多的列（如性别）。
- 复合索引遵循 “最左匹配”：查询条件需包含复合索引的第一个列才会生效。

### 六、视图（View）

视图是基于 SELECT 语句的虚拟表，不存储数据，仅存储查询逻辑，简化复杂查询。

```sql
-- 1. 创建视图
CREATE VIEW v_student_score AS
SELECT s.name, s.class, sc.subject, sc.score 
FROM student s
LEFT JOIN score sc ON s.id = sc.student_id;

-- 2. 使用视图（和普通表一样查询）
SELECT * FROM v_student_score WHERE class = 1;

-- 3. 修改视图
ALTER VIEW v_student_score AS
SELECT s.name, sc.subject, sc.score, c.course_name 
FROM student s
LEFT JOIN score sc ON s.id = sc.student_id
LEFT JOIN course c ON sc.course_id = c.id;

-- 4. 删除视图
DROP VIEW IF EXISTS v_student_score;
```

### 七、存储过程与函数（简单介绍）

#### 1. 存储过程

预编译的 SQL 集合，可带参数，批量执行 SQL，提升效率。

```sql
-- 创建存储过程：查询指定班级的学生人数
DELIMITER // -- 修改结束符（默认;，避免冲突）
CREATE PROCEDURE get_student_count(IN class_id INT, OUT count_num INT)
BEGIN
    SELECT COUNT(*) INTO count_num FROM student WHERE class = class_id;
END //
DELIMITER ; -- 恢复结束符

-- 调用存储过程
CALL get_student_count(1, @count);
SELECT @count; -- 查看结果

-- 删除存储过程
DROP PROCEDURE IF EXISTS get_student_count;
```

#### 2. 存储函数

返回单个值的存储程序，可直接用于 SELECT 语句。

```sql
-- 创建存储函数：计算学生总分
DELIMITER //
CREATE FUNCTION get_total_score(student_id INT) RETURNS INT
BEGIN
    DECLARE total INT;
    SELECT chinese + math + english INTO total FROM student WHERE id = student_id;
    RETURN total;
END //
DELIMITER ;

-- 使用存储函数
SELECT name, get_total_score(id) AS total FROM student;

-- 删除存储函数
DROP FUNCTION IF EXISTS get_total_score;
```

### gitea教程 

#### 创建数据库及专门用户

1. 创建 gitea 数据库（如果尚未创建）
   首先创建 gitea 专用数据库，并指定兼容的字符集（utf8mb4 支持全量 Unicode 字符，包括 emoji，适合 Gitea 使用）：
	```sql
     CREATE DATABASE IF NOT EXISTS gitea 
     DEFAULT CHARACTER SET utf8mb4 
     COLLATE utf8mb4_unicode_ci;
     # IF NOT EXISTS：避免重复创建数据库时报错
	```
     utf8mb4：MySQL 中真正的 UTF-8 字符集，比默认的 utf8 兼容性更好
2. 创建 gitea 管理用户（允许所有主机访问）
   创建用户并设置密码，% 表示允许该用户从任意主机（IP）访问 MySQL：
     ```sql
     -- 替换 your_strong_password 为你自己的安全密码
     CREATE USER 'gitea_user'@'%' IDENTIFIED BY 'your_strong_password';
     ```
     gitea_user：自定义的用户名（你也可以改成自己喜欢的名称）
     %：通配符，代表所有主机（包括本地、局域网、公网主机）
     密码建议：包含大小写字母、数字、特殊符号，避免简单密码（如 123456）
3. 授予用户对 gitea 数据库的全部权限
   给创建的用户授予 gitea 数据库的所有管理权限（足够支撑 Gitea 的运行）：
     ```sql
     GRANT ALL PRIVILEGES ON gitea.* TO 'gitea_user'@'%';
     ```
     ALL PRIVILEGES：包含查询、插入、修改、删除、建表等所有权限
     gitea.*：限定权限仅作用于 gitea 数据库下的所有表，避免权限过大
4. 刷新权限使配置生效
   MySQL 的权限配置需要手动刷新才能立即生效：
     ```sql
     FLUSH PRIVILEGES;
     ```
5. （可选）验证用户权限
   执行以下命令可查看创建的用户权限，确认配置正确：
	```sql
     SHOW GRANTS FOR 'gitea_user'@'%';
	```
   输出结果会显示该用户拥有gitea.*的全部权限，说明配置成功。
   重要注意事项
   如果你的 MySQL 服务器配置了bind-address（在 my.cnf/my.ini 中），需要确保该参数不是127.0.0.1（仅允许本地访问），建议改为0.0.0.0（允许所有 IP 访问），否则即使配置了%，外部主机仍无法连接。
   生产环境中，若不需要全网访问，可将%替换为具体的 IP / 网段（如192.168.1.%），提升安全性。
   MySQL 5.7 默认开启密码验证插件，确保设置的密码符合复杂度要求（否则会创建用户失败）

## PostgreSQL 简介

PostgreSQL 是世界上最先进的开源对象关系型数据库系统，拥有超过15年的活跃开发和经过验证的架构，在可靠性、数据完整性和正确性方面赢得了极高的声誉。

### 主要特性

- **ACID 兼容**：完全支持事务的原子性、一致性、隔离性和持久性
- **标准兼容**：支持大部分 SQL:2008 标准
- **扩展性强**：支持自定义数据类型、函数、操作符
- **多种编程接口**：支持 C/C++、Java、.NET、Python、Ruby、Go 等
- **丰富的数据类型**：包括整数、数值、布尔、字符、日期、时间戳、二进制大对象等

### 版本信息

- **最新版本**: PostgreSQL 18.1 / 17.7 / 16.11
- **推荐版本**: PostgreSQL 17 (稳定版，长期支持)
- **官方下载**: https://www.postgresql.org/download/

## Windows 平台安装

### 系统要求

- **操作系统**: Windows 10 64位 / Windows 11 64位
- **处理器**: Intel 或 AMD 64位处理器
- **内存**: 最少2GB，推荐4GB或更多
- **存储**: 至少100MB安装空间，数据存储需要额外空间
- **其他**: 需要 Microsoft Visual C++ Redistributable

### 安装步骤

#### 步骤 1: 下载安装程序

1. 访问 PostgreSQL 官方下载页面: https://www.postgresql.org/download/windows/
2. 选择最新版本（推荐 PostgreSQL 17）
3. 下载文件: `postgresql-17.x-x-windows-x64.exe`

#### 步骤 2: 运行安装向导

1. 双击下载的安装程序
2. 点击 **Next** 开始安装

#### 步骤 3: 选择安装目录

- 默认路径: `C:\Program Files\PostgreSQL\17\`
- 如需更改，点击文件夹图标选择自定义路径
- 点击 **Next**

#### 步骤 4: 选择安装组件

选择需要安装的组件（推荐全选）：

- ✅ **PostgreSQL Server**: 核心数据库引擎
- ✅ **pgAdmin 4**: Web 图形化管理工具
- ✅ **Stack Builder**: 扩展组件下载工具
- ✅ **Command Line Tools**: 命令行工具集（psql 等）

点击 **Next**

#### 步骤 5: 设置数据目录

- 默认路径: `C:\Program Files\PostgreSQL\17\data`
- 这是数据库存储数据文件的位置
- 点击 **Next**

#### 步骤 6: 设置超级用户密码

- **重要**: 设置 `postgres` 用户的密码
- 这是数据库的超级管理员账户
- 请使用强密码并妥善保管
- 点击 **Next**

#### 步骤 7: 设置端口号

- 默认端口: **5432**
- 如果该端口被占用，可以改为其他端口（如 5433）
- 记录所选端口号，后续连接需要用到
- 点击 **Next**

#### 步骤 8: 设置区域

- 选择数据库的区域设置（Locale）
- 推荐: `[Default locale]` 或 `Chinese (Simplified)_China.936`
- 点击 **Next**

#### 步骤 9: 开始安装

1. 检查安装设置摘要
2. 点击 **Next** 开始安装
3. 等待安装完成（约2-5分钟）
4. 点击 **Finish** 完成安装

### 验证安装

#### 方法一: 使用 psql 命令行

1. 打开 Windows 搜索，输入 `psql`
2. 打开 **SQL Shell (psql)**
3. 按提示输入连接信息：
   - Server: `localhost`
   - Database: `postgres`
   - Port: `5432`（或你设置的端口）
   - Username: `postgres`
   - Password: 输入你设置的密码

4. 连接成功后，输入以下命令验证版本：

```sql
SELECT version();
```

#### 方法二: 使用 pgAdmin 4

1. 在开始菜单找到 **pgAdmin 4**
2. 启动后会自动连接到本地 PostgreSQL 实例
3. 在左侧导航栏展开 **Servers** → **PostgreSQL 17**
4. 右键点击选择 **Properties** 查看服务器信息

### 环境变量配置（可选）

为了方便在命令行中使用 PostgreSQL 工具，可以添加环境变量：

1. 打开 **系统属性** → **高级** → **环境变量**

2. 在 **系统变量** 中找到 **Path**，点击编辑

3. 添加 PostgreSQL 的 bin 目录路径：

   ```
   C:\Program Files\PostgreSQL\17\bin
   ```

4. 点击确定保存

## 基础配置与初始化

### 配置文件位置

#### 不同系统的配置路径

| 操作系统         | 配置文件路径                                  |
| ---------------- | --------------------------------------------- |
| Ubuntu/Debian    | `/etc/postgresql/<version>/main/`             |
| CentOS/RHEL      | `/var/lib/pgsql/<version>/data/`              |
| Windows          | `C:\Program Files\PostgreSQL\<version>\data\` |
| macOS (EDB)      | `/Library/PostgreSQL/<version>/data/`         |
| macOS (Homebrew) | `/opt/homebrew/var/postgresql@<version>/`     |

配置文件主要包括：

- `postgresql.conf`: 主配置文件，控制服务器行为
- `pg_hba.conf`: 客户端认证配置文件
- `pg_ident.conf`: 身份映射文件

---

### postgresql.conf 基础配置

#### 1. 连接和监听设置

```conf
# 监听地址（允许远程连接）
listen_addresses = 'localhost,*'  # 监听所有接口
# listen_addresses = '192.168.1.100'  # 或指定IP

# 端口号
port = 5432

# 最大连接数
max_connections = 100

# 共享缓冲区（推荐设置为内存的25%）
shared_buffers = 256MB
```

#### 2. 内存设置

```conf
# 工作内存（每个查询操作使用）
work_mem = 4MB

# 维护工作内存（VACUUM, CREATE INDEX等）
maintenance_work_mem = 64MB

# 有效缓存大小（应设置为总内存的50-75%）
effective_cache_size = 1GB
```

#### 3. 日志设置

```conf
# 启用日志
logging_collector = on

# 日志目录
log_directory = 'log'

# 日志文件名
log_filename = 'postgresql-%Y-%m-%d_%H%M%S.log'

# 日志级别
log_statement = 'all'  # 记录所有语句（生产环境谨慎使用）
log_min_duration_statement = 1000  # 记录超过1秒的查询
```

#### 4. 性能调优

```conf
# 随机页面成本（SSD可设置为1.0）
random_page_cost = 1.1

# 并行工作进程
max_parallel_workers_per_gather = 2
max_parallel_workers = 8

# 自动清理
autovacuum = on
```

### pg_hba.conf 认证配置

#### 认证方法说明

| 方法          | 描述                 | 安全性 |
| ------------- | -------------------- | ------ |
| trust         | 无条件允许连接       | 低     |
| reject        | 无条件拒绝连接       | -      |
| scram-sha-256 | 密码认证（推荐）     | 高     |
| md5           | MD5密码认证          | 中     |
| peer          | 使用操作系统用户认证 | 中     |
| ident         | 使用 ident 服务认证  | 中     |

#### 配置示例

```conf
# TYPE  DATABASE        USER            ADDRESS                 METHOD

# 本地连接使用 peer 认证
local   all             all                                     peer

# IPv4 本地连接使用 scram-sha-256
host    all             all             127.0.0.1/32            scram-sha-256

# IPv6 本地连接使用 scram-sha-256
host    all             all             ::1/128                 scram-sha-256

# 允许特定IP段连接（使用密码认证）
host    all             all             192.168.1.0/24          scram-sha-256

# 允许特定用户从特定IP连接
host    mydb            myuser          192.168.1.100/32        scram-sha-256

# 拒绝特定IP
host    all             all             192.168.1.200/32        reject

# 复制连接
host    replication     replicator      192.168.1.0/24          scram-sha-256
```

#### 配置规则说明

每条规则包含5个字段：

1. **TYPE**: 连接类型（local, host, hostssl, hostnossl）
2. **DATABASE**: 数据库名（all, specific_db, comma-separated）
3. **USER**: 用户名（all, specific_user）
4. **ADDRESS**: 客户端地址（IP/CIDR, hostname）
5. **METHOD**: 认证方法

**重要**: 规则按顺序匹配，第一个匹配的规则生效。

## 用户管理与权限配置

### PostgreSQL 角色系统

PostgreSQL 使用 **角色（Role）** 系统进行权限管理：

- 角色可以是一个用户（有 LOGIN 权限）
- 角色可以是一个组（无 LOGIN 权限，用于组织权限）
- 角色可以继承其他角色的权限

#### 角色类型

| 角色类型 | LOGIN权限 | 用途                     |
| -------- | --------- | ------------------------ |
| 用户角色 | ✅         | 实际登录数据库的用户     |
| 组角色   | ❌         | 组织权限，供其他角色继承 |
| 超级用户 | ✅         | 拥有所有权限（谨慎使用） |

---

### 创建和管理用户

#### 1. 创建用户

**基本语法**：

```sql
CREATE USER username WITH PASSWORD 'password';
```

**带选项的用户创建**：

```sql
CREATE USER app_user 
WITH 
    PASSWORD 'secure_password'
    LOGIN
    CREATEDB
    NOCREATEROLE
    CONNECTION LIMIT 10
    VALID UNTIL '2025-12-31';
```

**CREATE USER vs CREATE ROLE**：

- `CREATE USER` 等价于 `CREATE ROLE ... WITH LOGIN`
- `CREATE ROLE` 默认没有 LOGIN 权限

#### 2. 修改用户

```sql
-- 修改密码
ALTER USER username WITH PASSWORD 'new_password';

-- 授予超级用户权限
ALTER USER username WITH SUPERUSER;

-- 撤销超级用户权限
ALTER USER username WITH NOSUPERUSER;

-- 重命名用户
ALTER USER username RENAME TO new_username;

-- 设置连接限制
ALTER USER username WITH CONNECTION LIMIT 5;
```

#### 3. 删除用户

```sql
-- 简单删除（如果用户不拥有任何对象）
DROP USER username;

-- 强制删除（先转移对象所有权）
REASSIGN OWNED BY username TO new_owner;
DROP OWNED BY username;
DROP USER username;
```

---

### 权限系统

#### 权限层级

PostgreSQL 的权限分为多个层级：

1. **数据库级别**: CONNECT, CREATE, TEMP/TEMPORARY
2. **模式级别**: USAGE, CREATE
3. **表级别**: SELECT, INSERT, UPDATE, DELETE, TRUNCATE, REFERENCES, TRIGGER
4. **列级别**: SELECT, INSERT, UPDATE, REFERENCES
5. **序列级别**: USAGE, SELECT, UPDATE
6. **函数级别**: EXECUTE

#### 授予权限

**基本语法**：

```sql
GRANT privilege_name ON object_name TO role_name;
```

**示例**：

```sql
-- 授予数据库连接权限
GRANT CONNECT ON DATABASE mydb TO app_user;

-- 授予模式使用权限
GRANT USAGE ON SCHEMA public TO app_user;

-- 授予表权限
GRANT SELECT, INSERT, UPDATE, DELETE ON TABLE users TO app_user;

-- 授予所有权限
GRANT ALL PRIVILEGES ON DATABASE mydb TO admin_user;

-- 授予列级权限
GRANT SELECT (id, name, email) ON TABLE users TO readonly_user;
```

#### 撤销权限

```sql
-- 撤销特定权限
REVOKE DELETE ON TABLE users FROM app_user;

-- 撤销所有权限
REVOKE ALL PRIVILEGES ON DATABASE mydb FROM username;

-- 撤销角色成员资格
REVOKE group_role FROM username;
```

---

### 常用角色配置示例

#### 1. 创建只读角色

```sql
-- 创建只读角色
CREATE ROLE readonly;

-- 授予数据库连接权限
GRANT CONNECT ON DATABASE mydb TO readonly;

-- 授予模式使用权限
GRANT USAGE ON SCHEMA public TO readonly;

-- 授予所有表的读取权限
GRANT SELECT ON ALL TABLES IN SCHEMA public TO readonly;

-- 设置默认权限（未来创建的表自动授权）
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT SELECT ON TABLES TO readonly;

-- 创建只读用户并分配角色
CREATE USER analyst WITH PASSWORD 'password';
GRANT readonly TO analyst;
```

#### 2. 创建读写角色

```sql
-- 创建读写角色
CREATE ROLE readwrite;

-- 授予数据库和模式权限
GRANT CONNECT, CREATE ON DATABASE mydb TO readwrite;
GRANT USAGE, CREATE ON SCHEMA public TO readwrite;

-- 授予所有表的读写权限
GRANT SELECT, INSERT, UPDATE, DELETE ON ALL TABLES IN SCHEMA public TO readwrite;

-- 授予序列权限
GRANT USAGE, SELECT ON ALL SEQUENCES IN SCHEMA public TO readwrite;

-- 设置默认权限
ALTER DEFAULT PRIVILEGES IN SCHEMA public 
    GRANT SELECT, INSERT, UPDATE, DELETE ON TABLES TO readwrite;
ALTER DEFAULT PRIVILEGES IN SCHEMA public 
    GRANT USAGE, SELECT ON SEQUENCES TO readwrite;

-- 创建应用用户
CREATE USER app_user WITH PASSWORD 'password';
GRANT readwrite TO app_user;
```

#### 3. 创建管理员角色

```sql
-- 创建管理员角色
CREATE ROLE admin_role;

-- 授予数据库管理权限
GRANT ALL PRIVILEGES ON DATABASE mydb TO admin_role;

-- 授予创建角色和数据库的权限
ALTER ROLE admin_role WITH CREATEROLE CREATEDB;

-- 创建管理员用户
CREATE USER db_admin WITH PASSWORD 'admin_password';
GRANT admin_role TO db_admin;
```

```sql

-- 查看特定角色的权限
\du rolename

-- 查看表的权限
\z tablename
```

### 最佳实践

1. **最小权限原则**: 只授予用户完成任务所需的最少权限
2. **使用组角色**: 通过角色继承管理权限，而不是直接授予用户
3. **定期审查**: 定期检查和清理不必要的权限
4. **强密码策略**: 为用户设置强密码，定期更换
5. **区分环境**: 开发、测试、生产环境使用不同的用户和权限
6. **避免超级用户**: 日常操作避免使用超级用户账户

## 常见问题与故障排除

### 问题 1: 连接被拒绝

**错误信息**：

```
psql: error: connection to server at "localhost" (::1), port 5432 failed: 
Connection refused
```

**可能原因和解决方案**：

1. **服务未启动**

   ```bash
   # 检查服务状态
   sudo systemctl status postgresql
   
   # 启动服务
   sudo systemctl start postgresql
   ```

2. **监听地址配置错误**

   - 检查 `postgresql.conf` 中的 `listen_addresses`
   - 确保包含 `localhost` 或 `*`

3. **端口错误**

   - 确认连接端口与配置一致
   - 检查防火墙设置

---

### 问题 2: 认证失败

**错误信息**：

```
psql: error: FATAL:  password authentication failed for user "postgres"
```

**解决方案**：

1. **忘记密码**

   ```bash
   # 切换到 postgres 用户
   sudo -u postgres psql
   
   # 在 psql 中重置密码
   \password postgres
   ```

2. **认证方法配置错误**

   - 检查 `pg_hba.conf` 中的认证方法
   - 本地连接可临时使用 `trust` 方法测试

3. **用户不存在**

   ```bash
   # 查看所有用户
   sudo -u postgres psql -c "\du"
   ```

---

### 问题 3: Peer 认证失败

**错误信息**：

```
psql: error: FATAL:  Peer authentication failed for user "myuser"
```

**原因**：

- 使用 `peer` 认证时，操作系统用户名必须与 PostgreSQL 用户名匹配

**解决方案**：

1. **使用 sudo 切换用户**

   ```bash
   sudo -u myuser psql
   ```

2. **修改认证方法**
   编辑 `pg_hba.conf`：

   ```conf
   local   all             all                                     scram-sha-256
   ```

   然后重新加载配置。

3. **使用密码认证**

   ```bash
   psql -U myuser -h localhost -W
   ```

---

### 问题 4: 权限不足

**错误信息**：

```
ERROR:  permission denied for relation tablename
```

**解决方案**：

```sql
-- 授予所需权限
GRANT SELECT, INSERT, UPDATE, DELETE ON TABLE tablename TO username;

-- 或授予所有权限
GRANT ALL PRIVILEGES ON TABLE tablename TO username;

-- 如果是模式权限问题
GRANT USAGE ON SCHEMA schema_name TO username;
```

**解决方案**：




### 问题 7: 磁盘空间不足

**错误信息**：

```

```

## 性能优化建议

### 硬件层面

1. **内存**
   - 至少 4GB，推荐 8GB 或更多
   - 数据库专用的服务器可配置 64GB 或更多

2. **存储**
   - 使用 SSD 替代机械硬盘
   - 将数据文件和 WAL 日志放在不同磁盘
   - 使用 RAID 10 提高性能和可靠性

3. **CPU**
   - 多核 CPU 有利于并行查询
   - PostgreSQL 能够有效利用多核

---

### 配置参数调优

#### 1. 内存相关参数

```conf
# 共享缓冲区（推荐为系统内存的25%）
shared_buffers = 2GB

# 工作内存（每个查询操作）
work_mem = 64MB

# 维护工作内存
maintenance_work_mem = 512MB

# 有效缓存大小（应设置为总内存的50-75%）
effective_cache_size = 6GB
```

#### 2. 并发和并行

```conf
# 最大连接数
max_connections = 200

# 并行工作进程
max_parallel_workers_per_gather = 4
max_parallel_workers = 8

# 后台工作进程
max_worker_processes = 8
```

#### 3. 写入和 WAL

```conf
# WAL 缓冲区
wal_buffers = 16MB

# 检查点配置
checkpoint_completion_target = 0.9
max_wal_size = 2GB
min_wal_size = 512MB

# 同步提交
synchronous_commit = on  # 数据安全优先
# synchronous_commit = off  # 性能优先（可能丢失少量数据）
```

#### 4. 查询规划器

```conf
# 随机页面成本（SSD设置为1.0-1.1）
random_page_cost = 1.1

# 顺序扫描成本
seq_page_cost = 1.0

# 启用查询并行
default_statistics_target = 100
constraint_exclusion = partition
```

### 数据库设计优化

#### 1. 索引优化

```sql
-- 创建合适的索引
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_orders_user_id ON orders(user_id);

-- 复合索引
CREATE INDEX idx_orders_user_date ON orders(user_id, created_at);

-- 部分索引
CREATE INDEX idx_active_users ON users(email) WHERE active = true;

-- 表达式索引
CREATE INDEX idx_lower_email ON users(LOWER(email));
```

#### 2. 表分区

```sql
-- 范围分区
CREATE TABLE orders_2025_01 PARTITION OF orders
    FOR VALUES FROM ('2025-01-01') TO ('2025-02-01');

-- 列表分区
CREATE TABLE users_active PARTITION OF users
    FOR VALUES IN (true);
```

```bash
# 手动执行清理
vacuumdb --all --analyze

# 全量清理（回收空间）
vacuumdb --all --full
```

### 监控和诊断

#### 1. 启用统计收集

```conf
# 在 postgresql.conf 中
track_activities = on
track_counts = on
track_functions = all
track_io_timing = on
```

#### 2. 常用监控查询

```sql
-- 查看慢查询
SELECT query, mean_time, calls, total_time
FROM pg_stat_statements
ORDER BY mean_time DESC
LIMIT 10;

-- 查看当前活动连接
SELECT * FROM pg_stat_activity;

-- 查看表统计
SELECT 
    schemaname,
    tablename,
    n_tup_ins,
    n_tup_upd,
    n_tup_del,
    n_live_tup,
    n_dead_tup
FROM pg_stat_user_tables;

-- 查看索引使用情况
SELECT 
    schemaname,
    tablename,
    indexname,
    idx_scan,
    idx_tup_read,
    idx_tup_fetch
FROM pg_stat_user_indexes;
```

#### 3. 使用 pg_stat_statements

```sql
-- 启用扩展
CREATE EXTENSION IF NOT EXISTS pg_stat_statements;

-- 查看最耗时的查询
SELECT 
    query,
    calls,
    total_time,
    mean_time,
    rows
FROM pg_stat_statements
ORDER BY total_time DESC
LIMIT 10;
```

### 备份和恢复

## 总结

本教程涵盖了 PostgreSQL 在所有主要平台的安装、配置、用户管理和常见问题解决：

- **Windows**: 使用官方安装程序，图形化界面，适合新手
- **macOS**: 四种安装方式，根据需求选择 EDB、Postgres.app、Homebrew 或 MacPorts
- **Linux**: 使用各发行版的包管理器，命令行安装

安装完成后，重点配置：

1. `postgresql.conf`: 性能参数、内存设置、日志配置
2. `pg_hba.conf`: 认证方法、远程访问控制
3. 用户权限: 遵循最小权限原则，使用角色管理

遇到问题时，首先查看日志文件，使用 `psql` 命令行工具测试基本连接，然后逐步排查网络、认证和权限配置。

PostgreSQL 是一个功能强大、稳定可靠的数据库系统，正确的安装和配置是发挥其性能的基础。希望本教程能帮助你顺利完成 PostgreSQL 的部署和配置。
