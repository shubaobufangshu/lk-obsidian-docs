# 代码检查与格式化工具

代码检查（Lint）与格式化为前端工程化核心：**代码检查**校验语法、逻辑与规范，提前拦截 Bug 及不规范写法；**格式化**统一缩进、引号、换行等风格，减少协作争议。ESLint 是前端代码检查的事实标准，常与 Prettier 等配合，构建质量保障体系。

## ESLint 

### 初始化生成新配置（推荐）

直接在项目根目录执行：

```bash
npx eslint --init
```

然后按提示选择：

- How would you like to use ESLint? → **To check syntax, find problems, and enforce code style**
- What type of modules does your project use? → 根据你的项目选（ESM/CommonJS）
- 框架：Vue / React / None
- TypeScript：按需选择
- 配置文件格式：**JavaScript**
- 是否安装依赖：y

执行完会**自动生成 `eslint.config.js`**，然后再运行：

```bash
npx eslint --fix
```

就正常了。

ESLint 是开源、可配置的 JS/TS 代码检查工具，通过规则校验提升质量、统一风格，支持 React/Vue/Angular 等框架，为现代前端工程化标配。

#### 1. 核心价值

- **拦截 Bug**：检测未定义变量、不合理条件、废弃 API 等，降低线上故障；
- **统一规范**：解决多人协作风格差异，减少评审争议；
- **适配演进**：原生支持 ES6+，通过插件/解析器适配 TS、React、Vue 等；
- **自定义规则**：针对业务场景定制（如禁止 `alert`、强制埋点）；
- **自动修复**：多数规则支持自动修复，减少手动调整成本。

#### 2. 核心概念

| 概念                | 说明                                                         |
| ------------------- | ------------------------------------------------------------ |
| 规则（Rules）       | 最小校验单元，支持三级：`off/0`（关）、`warn/1`（警告）、`error/2`（阻断） |
| 解析器（Parser）    | 将代码转为 AST，不同语法对应不同解析器：Espree（JS）、`@typescript-eslint/parser`（TS）、`vue-eslint-parser`（Vue） |
| 插件（Plugins）     | 扩展规则集，命名 `eslint-plugin-[name]`，使用时省略前缀（如 `react`） |
| 配置继承（Extends） | 复用规则集：官方（`eslint:recommended`）、插件（`plugin:vue/essential`）、共享配置（`eslint-config-airbnb`） |
| 环境（Env）         | 声明运行环境，避免全局变量报错（如浏览器 `window`、Node `process`） |

#### 3. 完整配置文件（.eslintrc.js）

以下是适配 React + TypeScript + Vue 混合项目的完整配置示例，覆盖绝大多数场景：

```javascript
module.exports = {
  // 根配置标识：避免 ESLint 向上查找父目录配置
  root: true,

  // 环境配置：声明代码运行的环境（多环境兼容）
  env: {
    browser: true, // 浏览器环境（window/document）
    es2024: true,  // 支持 ES2024 语法
    node: true,    // Node.js 环境（process/__dirname）
    jest: true,    // Jest 测试环境（describe/test）
  },

  // 全局变量：声明自定义全局变量（避免 no-undef 规则报错）
  globals: {
    __APP_VERSION__: "readonly", // 只读全局变量（如构建注入的版本号）
    Vue: "writable",             // 可写全局变量（如 CDN 引入的 Vue）
  },

  // 继承配置：多层继承，优先级从后到前
  extends: [
    "eslint:recommended",                // ESLint 官方推荐规则
    "plugin:react/recommended",          // React 基础规则
    "plugin:react-hooks/recommended",    // React Hooks 规则
    "plugin:@typescript-eslint/recommended", // TS 基础规则
    "plugin:vue/essential",              // Vue 基础规则
    "prettier",                          // 禁用 ESLint 与 Prettier 冲突的规则
  ],

  // 解析器：根据技术栈动态选择
  parser: process.env.USE_TS ? "@typescript-eslint/parser" : "vue-eslint-parser",

  // 解析器选项：精细化配置语法支持
  parserOptions: {
    ecmaVersion: "2024",                // 支持的 ES 版本
    sourceType: "module",               // 模块化方案（ES Module，CommonJS 设为 "script"）
    ecmaFeatures: {
      jsx: true,                        // 支持 JSX 语法（React/Vue JSX）
      tsx: true,                        // 支持 TSX 语法
    },
    // Vue 解析器专属配置：区分脚本/样式解析器
    parser: {
      js: "espree",
      ts: "@typescript-eslint/parser",
    },
  },

  // 插件列表：省略 eslint-plugin- 前缀
  plugins: ["react", "@typescript-eslint", "vue", "import"],

  // 自定义规则：覆盖继承的规则（优先级最高）
  rules: {
    // 基础 JS 规则
    "no-console": ["warn", { allow: ["warn", "error"] }], // 允许 console.warn/error，其余警告
    "no-debugger": "error",                               // 禁止 debugger（生产环境严格拦截）
    "semi": ["error", "always"],                          // 强制使用分号
    "quotes": ["error", "single"],                        // 强制单引号
    "indent": ["error", 2, { SwitchCase: 1 }],            // 缩进 2 空格，switch case 缩进 1 级
    "no-unused-vars": "off",                              // 关闭原生未使用变量规则（TS 版本更强大）

    // React 规则
    "react/prop-types": "off",                            // TS 项目无需 prop-types 校验
    "react-hooks/rules-of-hooks": "error",                // 强制 Hooks 规则（如只能在函数组件中使用）
    "react-hooks/exhaustive-deps": "warn",                // 检查 Hooks 依赖（警告级别，避免过度严格）

    // TypeScript 规则
    "@typescript-eslint/no-unused-vars": ["error", { argsIgnorePattern: "^_" }], // 忽略下划线开头的未使用参数
    "@typescript-eslint/explicit-module-boundary-types": "off", // 关闭显式导出类型（简化代码）
    "@typescript-eslint/no-explicit-any": "warn",         // any 类型设为警告（逐步收敛）

    // Vue 规则
    "vue/multi-word-component-names": "off",              // 关闭组件名多单词限制（小型项目简化）
    "vue/no-unused-components": "error",                  // 禁止未使用的 Vue 组件

    // 导入规则
    "import/order": ["warn", { 
      "newlines-between": "always", 
      groups: ["builtin", "external", "internal"] 
    }], // 规范导入顺序：内置模块 → 第三方模块 → 内部模块
  },

  // 覆盖配置：针对特定文件定制规则
  overrides: [
    {
      files: ["*.config.js", "*.config.ts"], // 配置文件（如 vite.config.ts）
      rules: { "no-console": "off" },        // 配置文件允许 console
    },
    {
      files: ["src/**/*.test.ts", "src/**/*.spec.ts"], // 测试文件
      rules: { "@typescript-eslint/no-unused-vars": "off" }, // 测试文件允许未使用变量
    },
  ],
};
```

### ESLint 与 Prettier 协同

ESLint 负责代码规范与逻辑校验，Prettier 专注格式化（缩进、引号、换行）。二者互补但规则可能冲突，需配置协同。

#### 1. 安装依赖

```bash
npm install prettier --save-dev               # Prettier 核心
npm install eslint-config-prettier --save-dev # 关闭 ESLint 中与 Prettier 冲突的规则
npm install eslint-plugin-prettier --save-dev # 将 Prettier 格式问题作为 ESLint 规则报出
```

#### 2. 配置 Prettier（.prettierrc.js）

```javascript
module.exports = {
  printWidth: 120,          // 每行最大字符数
  tabWidth: 2,              // 缩进空格数
  useTabs: false,           // 禁用制表符
  singleQuote: true,        // 单引号
  semi: true,               // 分号
  trailingComma: "es5",     // ES5 尾逗号（对象/数组）
  bracketSpacing: true,     // 对象括号两侧空格 { a: 1 }
  arrowParens: "avoid",     // 箭头函数单参数省略括号
  vueIndentScriptAndStyle: true, // Vue SFC 内 script/style 缩进
  endOfLine: "lf",          // 换行符 LF
};
```

#### 3. 集成到 ESLint（.eslintrc.js）

```javascript
module.exports = {
  extends: [
    // ... 原有配置
    "plugin:prettier/recommended" // 启用 prettier 插件 + 禁用冲突规则 + 格式化问题转 ESLint 错误
  ],
  rules: {
    // ... 原有规则
    "prettier/prettier": "error",  // 将 Prettier 格式问题标记为错误
  },
};
```

> **推荐工作流**：`prettier --write` 自动格式化 → `eslint --fix` 修复逻辑问题 → 提交前借助 `lint-staged` + `husky` 自动执行。

#### 4. Prettier 忽略文件（.prettierignore）

```
node_modules/
dist/
*.config.js
*.md            # 若无需格式化 Markdown
```

### 工程化集成：自动化校验与格式化

#### 1. 编辑器集成（VS Code）

安装 ESLint、Prettier 插件，配置 `settings.json` 实现保存时自动修复：

```json
{
  "editor.codeActionsOnSave": { "source.fixAll.eslint": true },
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.formatOnSave": true,
  "eslint.validate": ["javascript", "typescript", "vue", "jsx", "tsx"]
}
```

#### 2. Git 提交前校验（Husky + lint-staged）

增量校验本次提交代码，阻止不规范代码入库：

```bash
npm install husky lint-staged --save-dev
npx husky install
npm pkg set scripts.prepare="husky install"   # 自动启用钩子
npx husky add .husky/pre-commit "npx lint-staged"
```

`package.json` 配置：

```json
{
  "lint-staged": {
    "*.{js,jsx,ts,tsx,vue}": ["eslint --fix", "prettier --write"],
    "*.{json,md,css,scss}": ["prettier --write"]
  }
}
```

#### 3. 构建/CI 阶段集成

##### （1）Vite 集成

```bash
npm install vite-plugin-eslint --save-dev
```

```typescript
// vite.config.ts

```

### 常见问题与解决方案

#### 1. ESLint 不识别 TS/JSX/Vue 语法

- **原因**：缺少对应解析器/插件。
- **方案**：
  - **TS**：安装 `@typescript-eslint/parser` + `@typescript-eslint/eslint-plugin`，配置 `parser: "@typescript-eslint/parser"`。
  - **React JSX/TSX**：安装 `eslint-plugin-react`，配置 `parserOptions.ecmaFeatures.jsx: true`。
  - **Vue**：安装 `vue-eslint-parser` + `eslint-plugin-vue`，配置 `parser: "vue-eslint-parser"`。

#### 2. 自动修复不生效

- **原因**：规则不支持修复、编辑器配置错误、文件被忽略。
- **方案**：
  - 查阅 [ESLint 规则列表](https://eslint.org/docs/rules/) 确认 `--fix` 支持性。
  - 检查 VS Code 插件状态及 `settings.json` 中的 `"editor.codeActionsOnSave"`。
  - 手动执行 `npx eslint --fix <文件>` 验证，排除 `.eslintignore` 干扰。

#### 3. Prettier 与 ESLint 规则冲突

- **原因**：ESLint 开启了与 Prettier 重复的格式化规则（如 `indent`、`quotes`）。
- **方案**：在 ESLint `extends` 末尾添加 `plugin:prettier/recommended`（自动禁用冲突规则 + 集成格式化检查）。

#### 4. 全局变量报错（no-undef）

- **原因**：ESLint 未识别自定义全局变量。
- **方案**：
  - 在 `.eslintrc.js` 中声明 `globals: { __APP_VERSION__: "readonly" }`。
  - 或在文件顶部添加 `/* global __APP_VERSION__ */` 注释。

### 工具选型对比

| 工具        | 核心能力           | 适用场景            | 与 ESLint 的配合方式                                     |
| ----------- | ------------------ | ------------------- | -------------------------------------------------------- |
| Prettier    | 代码格式化         | 统一缩进/引号/换行  | `eslint-config-prettier` + `plugin:prettier/recommended` |
| TypeScript  | 静态类型检查       | TS 项目类型校验     | ESLint 负责语法风格，TS 负责类型                         |
| Stylelint   | CSS/SCSS/LESS 检查 | 样式文件规范        | 独立运行，可配合 Husky 并行校验                          |
| Commitlint  | Git 提交信息校验   | 规范 commit message | 配合 `commit-msg` 钩子，与 ESLint 互补                   |
| Lint-staged | 增量校验暂存区文件 | Git 提交前快速校验  | 调用 ESLint/Prettier 命令，避免全量检查                  |

---

### 最佳实践总结

1. **分层配置**：用 `root: true` 防止配置向上合并；用 `overrides` 区分业务代码、配置文件、测试文件的规则。
2. **最小规则集**：优先继承 `eslint:recommended` 等成熟集，只定制团队核心规则，避免过度配置。
3. **自动化三层保障**：编辑器保存时自动修复 → Git 提交前 `lint-staged` 增量校验 → CI 全量校验。
4. **规则渐进式落地**：老项目先设 `warn`，逐步修复后改 `error`；新项目直接启用严格规则。
5. **平衡规范与效率**：禁用无实际价值的严格规则（如 Vue 组件名单词限制），避免挫伤开发体验。
6. **共享配置包**：团队开发 `eslint-config-[团队名]`，统一多项目配置，减少重复维护。

## Prettier 
### 一、核心前提
Prettier 是**代码格式化工具**（管缩进、引号、换行、分号），ESLint 是**代码质量工具**（管语法错误、未使用变量），两者配合最佳；下面从0到1完整配置，适配你的 vitepress 项目。

### 二、第一步：安装 Prettier 核心包
在项目根目录（`D:\code\vitpress\knowlageapp`）执行：
```bash
# 安装到开发依赖
npm install prettier --save-dev
# 或 yarn/pnpm
# yarn add prettier --dev
# pnpm add prettier -D
```

### 三、第二步：创建 Prettier 配置文件（3种任选1）
在**项目根目录**新建以下任一文件（推荐 `.prettierrc.js`，最灵活）：
##### 方案1：.prettierrc.js（推荐，支持注释/动态配置）
```javascript
// .prettierrc.js
module.exports = {
  // 基础格式化规则（通用+适配vitepress）
  printWidth: 100,          // 每行最大字符数（避免过长换行）
  tabWidth: 2,              // 缩进2空格（前端标准）
  useTabs: false,           // 禁用Tab，用空格
  semi: true,               // 语句末尾加分号
  singleQuote: true,        // 用单引号（JS/TS标准）
  quoteProps: 'as-needed',  // 对象属性仅必要时加引号
  jsxSingleQuote: true,     // JSX用单引号
  trailingComma: 'es5',     // ES5允许的地方加尾逗号（数组/对象）
  bracketSpacing: true,     // 对象括号前后加空格 { foo: bar }
  arrowParens: 'always',    // 箭头函数参数始终加括号 (x) => x
  endOfLine: 'lf',          // 换行符统一为LF（跨平台兼容）
  embeddedLanguageFormatting: 'auto', // 自动格式化内嵌代码（如markdown里的代码块）
};
```

##### 方案2：.prettierrc（JSON格式，简洁）
```json
{
  "printWidth": 100,
  "tabWidth": 2,
  "useTabs": false,
  "semi": true,
  "singleQuote": true,
  "trailingComma": "es5",
  "bracketSpacing": true,
  "arrowParens": "always",
  "endOfLine": "lf"
}
```

##### 方案3：package.json 中添加（无需新建文件）
在 `package.json` 里加 `"prettier"` 字段：
```json
{
  "name": "knowlageapp",
  "devDependencies": {
    "prettier": "^3.3.3"
  },
  "prettier": {
    "printWidth": 100,
    "tabWidth": 2,
    "singleQuote": true,
    "trailingComma": "es5"
  }
}
```

### 四、第三步：创建忽略文件（.prettierignore）
必须建！避免格式化node_modules、dist、缓存等无用文件，根目录新建 `.prettierignore`：
```
# 忽略目录
node_modules/
dist/
build/
.vitepress/cache/
# 忽略文件
*.log
.DS_Store
```

### 五、第四步：基础使用命令（格式化整个项目）
```bash
# 1. 检查哪些文件需要格式化（不修改）
npx prettier --check .

# 2. 一键格式化所有文件（最常用，直接修复格式）
npx prettier --write .

# 3. 格式化指定文件/目录
npx prettier --write src/ docs/
npx prettier --write index.js README.md
```

### 六、第五步：与ESLint 兼容（关键！避免冲突）
你之前装了ESLint v10，两者规则会冲突（比如引号、分号），必须装**兼容插件**：
##### 1. 安装兼容依赖
```bash
# 解决ESLint与Prettier冲突，让ESLint优先遵循Prettier规则
npm install eslint-config-prettier eslint-plugin-prettier --save-dev
```

##### 2. 修改 ESLint 配置（eslint.config.js）
打开你之前生成的 `eslint.config.js`，加入Prettier配置，最终完整配置如下：
```javascript
// eslint.config.js
import js from '@eslint/js';
import prettier from 'eslint-plugin-prettier';

export default [
  // 基础JS规则
  js.configs.recommended,
  // 启用Prettier插件+关闭冲突规则
  {
    plugins: {
      prettier: prettier,
    },
    rules: {
      // 把Prettier格式问题当作ESLint警告/错误
      'prettier/prettier': 'error',
      // 关闭ESLint中与Prettier冲突的规则
      'no-unused-vars': 'warn',
      'no-undef': 'error',
    },
  },
];
```

##### 3. 验证兼容
```bash
# 同时执行ESLint检查+Prettier格式化
npx eslint . --fix
npx prettier --write .
```

### 七、第六步：添加npm脚本（方便后续使用）
在 `package.json` 的 `"scripts"` 里加：
```json
{
  "scripts": {
    "format": "prettier --write .",       // 一键格式化
    "format:check": "prettier --check .",  // 检查格式
    "lint": "eslint .",                   // ESLint检查
    "lint:fix": "eslint . --fix"          // ESLint自动修复
  }
}
```
后续直接用：
```bash
npm run format  # 格式化所有文件
npm run lint:fix  # ESLint修复
```

### 八、第七步：VSCode 自动格式化（可选，开发必备）
1. 安装VSCode插件：**Prettier - Code formatter**
2. 打开VSCode设置（Ctrl+,），搜索并设置：
   - `Default Formatter` → 选 `Prettier - Code formatter`
   - 勾选 `Format On Save`（保存时自动格式化）
3. 项目根目录新建 `.vscode/settings.json`（推荐，团队统一）：
```json
{
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.formatOnSave": true,
  "editor.formatOnPaste": true
}
```

### 常见问题
1. 报错「找不到配置」：确保 `.prettierrc.js` 在**项目根目录**
2. ESLint和Prettier冲突：必须装 `eslint-config-prettier` + 改 `eslint.config.js`
3. 不想格式化某文件：在文件顶部加 `/* prettier-ignore */`
