# C++ 基础部分知识点

## 一、语言认知与编译基础
### 1. 核心特性与Python本质差异
- **语言类型**：C++是**编译型静态强类型**语言，代码需先编译为机器码再执行，编译期完成类型检查；Python是解释型动态强类型语言，逐行解释执行，运行期推断类型。
- **内存管理**：C++由开发者手动管理堆内存（申请/释放），无自动垃圾回收；Python通过引用计数+垃圾回收自动管理内存。
- **执行效率**：C++直接运行机器码，性能远高于Python，适合底层、高性能场景。

### 2. 编译执行流程
源代码（.cpp）→ 预处理（处理#include、#define等）→ 编译（生成汇编代码.s）→ 汇编（生成目标文件.o/.obj）→ 链接（合并库文件，生成可执行程序）

### 3. 标准与环境
- 主流标准：C++98（传统标准）、C++11（现代C++起点，基础部分以此为基准）、C++14/17/20/23
- 编译器：GCC、Clang、MSVC；IDE：Visual Studio、CLion、VS Code

### 4. 基础输入输出
- 头文件：`<iostream>`，所有标准库组件位于 `std` 命名空间
- 输出：`std::cout << 内容 << std::endl;`（endl=换行+刷新缓冲区）
- 输入：`std::cin >> 变量;`
- 示例框架
```cpp
#include <iostream>  // 包含标准输入输出头文件
using namespace std; // 展开std命名空间，简化代码（大型项目不推荐）

int main() { // 程序唯一入口，返回int类型
	int a = 0;        
	cin >>a
    cout << "Hello World" << endl;
    return 0;        // 0表示正常退出
}
```
> 语法差异：C++以分号`;`作为语句结束符，用大括号`{}`划分代码块；Python以缩进划分代码块，无分号强制要求。

## 二、数据类型与变量
### 1. 静态类型规则
变量必须**先声明类型再使用**，编译期校验类型合法性，类型不可动态变更；Python变量是对象标签，无固定类型。

### 2. 基本内置类型
| 类型分类 | 具体类型                                            | 说明                                    |
| ---- | ----------------------------------------------- | ------------------------------------- |
| 整型   | `short` / `int` / `long` / `long long`          | 字节数随平台变化，默认带符号；加`unsigned`变为无符号型      |
| 浮点型  | `float`（单精度） / `double`（双精度，默认） / `long double` | 浮点运算优先用double，精度更高                    |
| 字符型  | `char`                                          | 本质是1字节整型，存储ASCII值                     |
| 布尔型  | `bool`                                          | 取值`true`/`false`，非0值自动转为true，0转为false |

### 3. 变量初始化
1. 赋值初始化：`int a = 10;`
2. 括号初始化：`int b(10);`
3. **列表初始化（C++11推荐）**：`int c{10};`，禁止隐式缩窄转换（如int转char）

### 4. 常量
- **宏常量**：`#define MAX 100`，预处理阶段纯文本替换，无类型检查，不推荐
- **const常量**：`const int a = 10;`，编译期类型检查，有作用域，推荐使用
- **constexpr常量表达式**：`constexpr int b = 20;`，编译期计算值，性能更优

### 5. 类型转换
- 隐式转换：编译器自动完成（如int转double），遵循“低精度向高精度转换”规则
- 显式转换
  - C风格：`(int)3.14`，不推荐，语义不清晰
  - C++推荐：`static_cast<int>(3.14)`，编译期检查，可读性强

## 三、运算符与流程控制
### 1. 核心运算符
- 算术运算符：`+ - * / % ++ --`
  > 注意：C++中**整数/整数结果仍为整数**（截断小数）；Python3中`/`返回浮点数，`//`才是整数除法。
- 关系运算符：`== != > < >= <=`
- 逻辑运算符：`&& || !`，遵循短路求值规则，与Python一致
- 三目运算符：`条件 ? 表达式1 : 表达式2`
  > C++特性：三目运算符返回的是变量引用，可作为左值赋值，例如 `a > b ? a : b = 100;`

### 2. 分支结构
- `if / else if / else`：语法逻辑与Python一致，用大括号包裹代码块
- `switch`：仅支持整型、字符型、枚举类型匹配；case默认穿透，需`break`跳出；Python3.10+的match支持更多类型

### 3. 循环结构
1. `while` / `do-while`：do-while先执行一次循环体再判断条件
2. **传统for循环**：`for(初始化语句; 循环条件; 更新语句)`，基于索引控制，与Python的for-in迭代逻辑不同
3. **范围for（C++11）**：`for(auto x : 容器)`，遍历序列元素，类似Python的`for x in list`
4. `break` / `continue`：作用与Python完全一致

## 四、函数基础
### 1. 定义与声明
- 定义格式：`返回值类型 函数名(参数列表) { 函数体 }`
  - 无返回值用`void`，必须显式声明返回类型；Python用`def`定义，无显式返回类型
- 函数声明：函数定义在调用之后时，需提前声明函数签名，告知编译器函数存在
  ```cpp
  int add(int a, int b); // 声明
  int main() { add(1,2); }
  int add(int a, int b) { return a+b; } // 定义
  ```

### 2. 参数传递
- **值传递**：实参拷贝一份给形参，函数内修改形参不影响实参
  > 对比Python：Python不可变对象传参等价于值传递；可变对象传参传递的是对象引用，函数内修改会影响原对象。
- 地址传递（指针）、引用传递：见后续指针/引用章节，可在函数内修改实参

### 3. 进阶特性
- **默认参数**：从右向左依次设置默认值，声明中指定默认值，定义中不能重复写
- **函数重载（C++特有）**：同一作用域下，函数名相同、**参数列表不同**（个数/类型/顺序不同），与返回值无关
  - 原理：编译期根据参数列表对函数名进行修饰（名字粉碎），生成不同符号
  - Python不支持原生函数重载，后定义的函数会覆盖前者
- 占位参数：`void func(int)`，调用时必须传参，无实际变量名，用于运算符重载等拓展场景

## 五、数组与字符串
### 1. 一维数组
- 定义：`类型 数组名[数组长度];`，长度必须是**编译期常量**，不能是变量（变长数组VLA是GCC扩展，非标准）
- 初始化：`int arr[5] = {1,2,3};`，未完全初始化的剩余元素默认置0（全局数组默认全0，局部数组默认随机值）
- 核心特性：内存连续、元素同类型；数组名是首元素的地址常量，不可修改
- 对比Python列表：Python列表动态扩容、可存不同类型元素；C++静态数组长度固定、类型统一。

### 2. 二维数组
- 定义：`int arr[2][3];`（2行3列）
- 内存布局：按行连续存储
- 初始化：`int arr[2][3] = {{1,2,3}, {4,5,6}};`

### 3. C风格字符串
- 本质：以`'\0'`（空字符）结尾的char数组
- 常用函数（头文件`<cstring>`）：`strlen`（求长度，不计'\0'）、`strcpy`（拷贝）、`strcat`（拼接）、`strcmp`（比较）

### 4. C++ string类（推荐使用）
- 头文件`<string>`，属于`std`命名空间，自动管理内存，安全便捷
- 核心操作
  - 拼接：`+` / `+=`
  - 长度：`size()` / `length()`
  - 查找：`find(子串)`，返回起始下标，找不到返回`string::npos`
  - 截取：`substr(起始位置, 长度)`
> 对比Python：string是可变对象（可修改单个字符），Python字符串是不可变对象。

## 六、指针基础（核心难点）
### 1. 基本概念
指针是存储**内存地址**的变量；Python无显式指针，变量均为对象的引用（语义上类似指针但不可直接操作地址）。

### 2. 基础操作
- 定义：`类型* 指针名;` 例如 `int* p;`
- 取地址`&`：`p = &a;` 获取变量a的内存地址
- 解引用`*`：`*p = 20;` 通过指针操作指向的内存空间

### 3. 指针大小
32位系统下占4字节，64位系统下占8字节，**与指向的数据类型无关**。

### 4. 特殊指针
- **空指针**：`nullptr`（C++11推荐，类型安全）、`NULL`、`0`；空指针不可解引用，会触发程序崩溃
- 野指针：指向非法/未初始化内存的指针，无任何有效指向，必须避免出现

### 5. const修饰指针
1. 指向常量的指针：`const int* p` → 指向的值不可改，指针的指向可改
2. 常量指针：`int* const p` → 指针的指向不可改，指向的值可改
3. 常指针常量：`const int* const p` → 指向和值都不可改

### 6. 指针与数组
- 数组名是首元素的地址常量，`arr` 等价于 `&arr[0]`
- 指针偏移：`p+1` 指向下一个同类型元素，偏移步长由指针指向的类型决定
- 数组下标访问本质：`arr[i]` 等价于 `*(arr+i)`

### 7. 指针与函数
指针作为函数参数，实现**地址传递**，可在函数内部修改实参的值，本质是把变量地址拷贝给形参。

## 七、引用基础
### 1. 基本概念
给已存在的变量起一个**别名**，操作引用完全等价于操作原变量。
- 定义：`类型& 别名 = 原变量;` 例如 `int& ref = a;`

### 2. 核心规则
- 必须在定义时初始化，不能存在空引用
- 初始化后不能再指向其他变量
- 引用本质：编译器底层实现为**指针常量**，自动完成解引用，语法更简洁

### 3. 常用场景
1. **引用做函数参数**：实现引用传递，效果等同地址传递，语法更简洁，是C++主流传参方式
2. 引用做函数返回值：禁止返回局部变量的引用（栈内存函数结束后释放），可返回静态变量/全局变量的引用

## 八、结构体与枚举
### 1. 结构体 struct
- 作用：自定义聚合数据类型，可包含不同类型的成员变量
- 定义与实例化
  ```cpp
  struct Student {
      string name;
      int age;
  };
  Student s1 = {"张三", 18}; // 实例化
  s1.age = 19;               // . 访问成员
  ```
- 结构体指针：用`->`访问成员，例如 `p->name`
- 可作为函数参数，支持值传递、地址传递、引用传递
> 对比Python：类似`dataclass`，但C++结构体可直接定义成员函数、支持访问控制，本质与class几乎一致（默认访问权限不同）。

### 2. 枚举 enum
- 作用：限定变量的取值范围，枚举值是编译期整型常量
- 定义：`enum Color { red, green, blue };`，默认从0开始递增，可手动赋值
- 限制：枚举变量只能赋值枚举值，不能直接赋值整数

## 九、内存管理基础
### 1. 内存分区模型
1. **代码区**：存放函数二进制指令，只读、共享
2. **全局区**：存放全局变量、静态变量、字符串常量，程序结束后由系统释放
3. **栈区**：由编译器自动分配释放，存放局部变量、函数参数，空间小、分配速度快
4. **堆区**：由开发者手动申请/释放，空间大，程序结束后系统回收
> 对比Python：Python解释器封装了内存管理，开发者无需感知栈堆差异，由垃圾回收器自动回收不再使用的对象。

### 2. new / delete 操作符
C++通过`new`申请堆内存，`delete`释放堆内存，必须配对使用，否则会造成内存泄漏。
- 申请单个变量：`int* p = new int(10);` → 释放：`delete p;`
- 申请数组：`int* arr = new int[10];` → 释放：`delete[] arr;`

## 十、其他基础语法
1. **命名空间**：解决命名冲突，`namespace 名称 { 代码 }`；`using namespace std;` 展开命名空间（简化代码，大型工程不推荐全局展开）
2. **文件结构**：`.h`头文件存放声明，`.cpp`源文件存放实现；通过`#include "头文件名"`引入，对应Python的`import`模块机制
3. **类型别名**：`typedef int myInt;` 给类型起别名，简化复杂类型写法
4. **auto关键字（C++11）**：编译期自动推导变量类型，必须初始化；本质仍是静态类型，与Python动态类型有本质区别

# C++ 进阶部分知识点
> 定位：聚焦C++核心编程范式与标准库体系，覆盖面向对象、泛型编程、STL、异常与流操作等核心能力，重点拆解深浅拷贝、多态原理、模板机制等核心疑难点。

## 一、面向对象编程（OOP）核心
### 1. 类与封装
- **class与struct的本质区别**：默认访问权限不同。`class`默认私有（private），`struct`默认公有（public），其余特性完全一致。
- 访问控制修饰符
  - `public`：类内、类外均可访问
  - `private`：仅类内可访问，子类也不可直接访问
  - `protected`：类内、子类可访问，类外不可访问
- 封装原则：成员属性设为`private`，通过`public`的get/set接口对外暴露，实现权限控制与数据校验。
- **this指针**：成员函数隐含的指针，指向调用该函数的对象本身。
  - 作用：解决形参与成员变量重名问题；支持链式编程（返回`*this`）
  - 本质：编译器自动传入的参数，不占用对象内存空间
- 静态成员
  - 静态成员变量：`static`修饰，类内声明、类外初始化，所有对象共享同一份数据，编译阶段分配内存
  - 静态成员函数：只能访问静态成员变量，不属于具体对象，可通过类名直接调用
- 友元：授予外部函数/类访问类私有成员的权限，破坏封装性，慎用。
  - 分类：全局函数做友元、类做友元、成员函数做友元

### 2. 构造与析构函数
- 构造函数：对象初始化时自动调用，函数名与类名一致，无返回值，支持重载。
  - 分类：默认无参构造、有参构造、拷贝构造函数
  - 编译器默认行为：未自定义时，自动提供空实现的无参构造、逐字节拷贝的拷贝构造
- 析构函数：对象销毁前自动调用，格式为`~类名()`，无参数、无返回值、不可重载，用于释放堆内存等资源。
- 调用时机：栈对象离开作用域时自动析构；堆对象执行`delete`时触发析构。
- **初始化列表**：`类名(参数) : 属性1(值1), 属性2(值2) {}`
  - 优势：直接初始化，比构造函数内赋值效率更高
  - 必须场景：初始化const成员变量、引用成员变量

### 3. 深浅拷贝与赋值重载（核心难点）
- 拷贝构造调用场景：用已有对象初始化新对象、对象值传递作为函数参数、函数值返回对象
- **浅拷贝**：编译器默认的逐字节内存拷贝。
  - 问题：当类中有堆区成员时，多个对象指向同一块堆内存，析构时会触发重复释放，导致程序崩溃。
- **深拷贝**：自定义拷贝构造函数，在堆区重新申请独立内存，拷贝原对象的堆数据，从根本上解决浅拷贝问题。
- 赋值运算符重载：解决**已有对象之间的赋值**问题，同样需要处理深拷贝。
  - 标准实现：返回自身引用（支持链式赋值），先判断是否为自身赋值，释放原有堆内存，再申请新内存拷贝数据。

## 二、运算符重载
- 本质：对已有运算符重新定义功能，本质是函数重载，函数名为`operator运算符`
- 核心规则：不能创造新运算符、不能改变优先级与结合性、至少一个操作数为自定义类型
- 两种实现方式
  1. 成员函数重载：隐含this指针作为左操作数
  2. 全局函数重载：需访问私有成员时配合友元
- 常用运算符重载
  - 算术运算符（+、-等）：实现自定义对象的运算逻辑
  - 左移运算符（<<）：只能用全局函数重载，配合友元实现自定义对象的标准输出
  - 递增递减运算符（++、--）：前置重载返回引用，后置重载用int占位参数区分，返回临时值
  - 函数调用运算符（()）：重载后对象称为**仿函数**，使用方式类似函数调用，STL中广泛使用
- 对比Python：Python通过`__add__`等魔术方法实现运算符重载，机制类似，但C++支持的运算符更全，类型检查更严格。

## 三、继承与多态
### 1. 继承基础
- 语法：`class 子类 : 继承方式 父类`
- 继承方式与访问权限变化

| 继承方式 | 父类public成员 | 父类protected成员 | 父类private成员 |
|---------|--------------|-----------------|---------------|
| public继承 | 子类public | 子类protected | 不可直接访问 |
| protected继承 | 子类protected | 子类protected | 不可直接访问 |
| private继承 | 子类private | 子类private | 不可直接访问 |

- 对象模型：子类包含父类的全部成员（私有成员被继承但不可直接访问），内存布局上父类成员在前，子类成员在后。
- 构造与析构顺序：构造先父后子，析构先子后父。

### 2. 多继承与菱形继承
- 多继承：一个子类同时继承多个父类，易出现命名冲突与设计混乱。
- 菱形继承问题：两个父类继承自同一个祖父类，子类会保留两份祖父类成员，造成数据冗余与访问二义性。
- 解决方案：**虚继承**（`virtual`修饰继承方式），子类只共享一份祖父类成员，通过虚基类指针访问公共成员。

### 3. 多态（核心难点）
- 分类
  - 静态多态：编译期确定函数地址，如函数重载、运算符重载
  - 动态多态：运行期确定函数地址，通过虚函数实现
- 动态多态三要素
  1. 存在继承关系
  2. 子类重写父类的虚函数
  3. 父类指针/引用指向子类对象
- 虚函数：`virtual`修饰的成员函数，子类重写时`virtual`关键字可省略。
- 纯虚函数与抽象类
  - 纯虚函数：`virtual 返回值类型 函数名() = 0;`
  - 抽象类：包含纯虚函数的类，无法实例化对象，子类必须重写所有纯虚函数才能实例化
- **底层实现原理**
  - 类中存在虚函数时，会生成一张**虚函数表（vtable）**，存储本类所有虚函数的地址
  - 对象实例化时，会自带一个**虚指针（vptr）**，指向本类的虚函数表
  - 函数调用时，运行时通过vptr查找vtable，确定最终调用的函数地址，实现晚绑定
- 对比Python：Python基于鸭子类型实现多态，只要对象有对应方法即可调用，无需继承与虚函数；C++是静态语言的显式多态，类型安全更有保障。

### 4. RTTI 运行时类型识别
- `dynamic_cast`：安全的向下转型，仅支持多态类型，转型失败返回`nullptr`，比强制转换安全
- `typeid`：获取对象的类型信息，返回`type_info`对象，可判断两个对象运行时是否为同一类型

## 四、C++ 强制类型转换进阶
C++提供四种语义明确的显式转换运算符，替代不安全的C风格强制转换，编译期检查更严格。
1. `static_cast`：基础类型转换、父子类指针/引用转换（无运行时检查，向下转型不安全）、`void*`互转
2. `dynamic_cast`：仅用于多态类型的父子类转换，运行时检查安全性，向下转型失败返回空指针
3. `const_cast`：专门用于修改`const`属性，增加或去除常量限定
4. `reinterpret_cast`：底层二进制重解释，指针转整数、不同类型指针互转，极度危险，非底层开发慎用

## 五、模板与泛型编程
- 泛型编程：不依赖具体数据类型编写通用代码，大幅提升代码复用性，C++通过模板实现。

### 1. 函数模板
- 语法：`template<typename T> 函数声明`（`typename`可替换为`class`）
- 调用方式
  1. 自动类型推导：编译器根据实参自动推导T的类型
  2. 显式指定类型：`函数名<类型>(参数)`
- 注意事项：自动推导不支持隐式类型转换；显式指定类型支持
- 重载规则：同名时优先调用普通函数，可通过`函数名<>(参数)`强制调用模板

### 2. 类模板
- 语法：`template<class T1, class T2> 类声明`
- 特点：只能显式指定类型，无法自动推导；支持设置默认模板参数
- 成员函数创建时机：普通类成员函数编译时生成，类模板成员函数调用时才生成

### 3. 模板特化
- 全特化：对特定类型做特殊实现，针对全部模板参数
- 偏特化：部分参数特化，或对某一类类型做范围特化

### 4. 分离编译问题
- 问题：模板声明与实现分离在.h和.cpp文件时，会出现链接错误。原因是模板编译期实例化，实现文件无法感知调用的类型，无法生成对应代码。
- 解决方案：将模板实现写入头文件；或在实现文件中显式实例化所需类型。

## 六、STL 标准模板库核心
STL六大组件：容器、算法、迭代器、仿函数、适配器、空间配置器，核心逻辑是**容器存储数据，算法操作数据，迭代器作为二者的桥梁**。

### 1. 序列式容器
| 容器 | 底层结构 | 核心特点 | 适用场景 |
|-----|---------|---------|---------|
| vector | 动态数组，内存连续 | 随机访问快，尾插尾删快，中间插入删除慢，自动扩容 | 频繁随机访问、尾部操作 |
| deque | 分段连续数组+中控器 | 两端插入删除快，支持随机访问，性能弱于vector | 双端队列场景 |
| list | 双向循环链表 | 任意位置插入删除快，不支持随机访问 | 频繁中间插入删除 |
- 核心注意点：vector扩容时会重新申请更大内存（GCC为2倍，VS为1.5倍），拷贝原数据后释放旧内存，会导致原有迭代器全部失效。

### 2. 关联式容器
| 容器 | 底层结构 | 核心特点 |
|-----|---------|---------|
| set / multiset | 红黑树 | 元素自动排序，set元素唯一，multiset允许重复，查找效率O(logn) |
| map / multimap | 红黑树 | 键值对存储，key自动排序，map key唯一，支持`[]`访问，multimap允许重复key |
- 常用操作：`insert`插入、`find`查找、`count`统计个数、`erase`删除

### 3. 容器适配器
- `stack`：栈，先进后出，底层默认用deque实现
- `queue`：队列，先进先出，底层默认用deque实现
- `priority_queue`：优先队列，底层堆实现，默认大顶堆

### 4. 迭代器
- 分类（功能从弱到强）：输入迭代器、输出迭代器、前向迭代器、双向迭代器、随机访问迭代器
- 迭代器失效：容器插入、删除、扩容操作可能导致迭代器失效，操作后需重新获取有效迭代器。

### 5. 常用算法
- 头文件：`<algorithm>`，数值算法头文件`<numeric>`
- 遍历：`for_each`、`transform`
- 查找：`find`、`find_if`、`binary_search`（仅有序序列）
- 排序：`sort`、`reverse`、`merge`
- 拷贝替换：`copy`、`replace`、`swap`
- 数值：`accumulate`、`fill`

## 七、异常处理
- 基本语法：`throw`抛出异常，`try`包裹可能异常的代码块，`catch`捕获对应类型的异常
- 栈解旋：异常抛出后，从try块到throw语句之间的所有栈对象会自动调用析构函数释放资源
- 标准异常库：头文件`<stdexcept>`，所有标准异常均继承自`exception`基类，如`runtime_error`、`out_of_range`等
- 异常规范：函数后可声明可抛出的异常类型，C++11后逐步弱化
- 对比Python：核心机制一致，但C++异常无默认栈追踪输出，异常类型检查更严格，性能开销略低。

## 八、文件与流操作
- IO流体系：标准IO（iostream）、文件IO（fstream）、字符串流（sstream）

### 1. 文件操作
- 头文件：`<fstream>`
- 三类文件流：`ifstream`（只读）、`ofstream`（只写）、`fstream`（读写）
- 打开模式：`ios::in`读、`ios::out`写、`ios::app`追加、`ios::binary`二进制
- 读写方式
  - 文本模式：通过`<<`、`>>`运算符格式化读写
  - 二进制模式：通过`read()`、`write()`成员函数按字节读写
- 状态判断：`is_open()`判断文件是否打开成功

### 2. 字符串流 stringstream
- 头文件：`<sstream>`
- 用途：实现字符串与数值类型的相互转换、字符串安全拼接，替代不安全的C风格字符串函数。
